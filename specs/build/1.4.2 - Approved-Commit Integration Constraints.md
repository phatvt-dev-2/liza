# Approved-Commit Integration Constraints

## User Story

As a reviewer/supervisor, I want merge operations constrained to approved commits so integration remains deterministic.

## Acceptance Criteria

- [ ] Integration operations only merge commits explicitly approved through review flow.
- [ ] Merge execution validates expected commit identity before integration.
- [ ] Divergence between approved commit and merge target is surfaced as an explicit failure.
- [ ] Integration constraints are enforceable in both CLI and supervisor/MCP paths.
