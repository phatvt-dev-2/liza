# Multi-Agent Coordination

## Goal

Coordinate planner, coder, and reviewer agents through shared blackboard state so work distribution and wake behavior are deterministic and observable.

## User Stories

- 1.2.1 (implemented) — As a multi-agent operator, I want blackboard coordination and wake/claim flow so roles synchronize without hidden message channels.
- 1.2.2 (planned) — As an operator, I want explicit stale-lease reclamation policy so abandoned claims do not block throughput.
- 1.2.3 (planned) — As a maintainer, I want coordination observability views so role-state drift is visible quickly.

## Success Criteria

- Agent coordination state is mediated through blackboard reads/modifies, not direct peer messaging.
- Planner wake triggers are evaluated in deterministic priority order.
- Coder and reviewer claim logic selects highest-priority eligible tasks under concurrency-safe mutation.
- Role status updates (idle/working/reviewing/planning) and lease fields reflect active assignments.
- Coordination issues become visible in shared state and logs for operator intervention.
