# Skills

## Goal

Provide a modular protocol library so agents can apply domain-specific methods (debugging, testing, review, architecture, backfill, delegation) while remaining inside contract constraints.

## User Stories

- 1.6.1 (implemented) — As an agent, I want explicit skill triggers so I can switch to the right protocol when task context changes.
- 1.6.2 (planned) — As a maintainer, I want skills packaged and distributed consistently so behavior is reproducible across sessions.
- 1.6.3 (planned) — As a collaborator, I want mandatory high-risk skills (debugging/testing/review) so critical work follows a repeatable methodology.

## Success Criteria

- Skills are stored as modular `SKILL.md` units under `skills/` and deployed through setup/bootstrap flow.
- Agent prompting and mode contracts expose skills and trigger rules for explicit invocation.
- Domain-critical workflows (debugging, testing, code review, architecture) have dedicated skills and are referenced by contract triggers.
- Skills are additive methodology layers: they cannot bypass contract gates or Tier 0 invariants.
