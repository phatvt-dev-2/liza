# Secrets and Recovery Protocols

## User Story

As a security-conscious maintainer, I want explicit secrets and recovery protocols so failures are contained and auditable.

## Acceptance Criteria

- [ ] Secret-handling rules prevent disclosure in logs, diffs, and outputs.
- [ ] Credential-file access requires explicit authorization workflow.
- [ ] Recovery protocols define deterministic response to violations and source/tool failures.
- [ ] Security and recovery actions are auditable and reproducible.
