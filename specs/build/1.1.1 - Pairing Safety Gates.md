# Pairing Safety Gates

## User Story

As a pairing user, I want hard invariants and approval gates so that agent state changes are trustworthy and auditable.

## Acceptance Criteria

- [ ] Pairing mode requires explicit human approval before any state-changing execution.
- [ ] Tier 0 invariants are enforced in pairing sessions with no bypass path.
- [ ] Agent proposals in pairing mode include intent, scope, risk, and validation details proportional to change complexity.
- [ ] Violations trigger explicit reset protocol rather than silent continuation.

## Implementation

- `contracts/CORE.md` — tier system, hard invariants, state machine, recovery protocol.
- `contracts/PAIRING_MODE.md` — pairing gate semantics and approval request standard.
- `contracts/AGENT_TOOLS.md` — tool policy that constrains execution behavior.
