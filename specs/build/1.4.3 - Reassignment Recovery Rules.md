# Reassignment Recovery Rules

## User Story

As a planner, I want reassignment rules (reuse vs fresh worktree) so rejected or failed tasks recover predictably.

## Acceptance Criteria

- [ ] Reassignment policy defines when existing worktree is reused versus recreated.
- [ ] Policy distinguishes rejection recovery from integration-failure recovery.
- [ ] Reassignment preserves required task history and audit fields.
- [ ] Recovery behavior remains deterministic under concurrent claims.
