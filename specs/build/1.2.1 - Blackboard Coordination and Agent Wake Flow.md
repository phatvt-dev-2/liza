# Blackboard Coordination and Agent Wake Flow

## User Story

As a multi-agent operator, I want blackboard-driven wake and claim behavior so planner, coder, and reviewer agents coordinate predictably.

## Acceptance Criteria

- [ ] Planner wake detection applies explicit trigger priority (initial planning, blocked, integration failure, hypothesis exhaustion, immediate discovery, sprint complete).
- [ ] Coder/reviewer claim operations select highest-priority eligible tasks with race-safe state updates.
- [ ] Supervisor loop updates role status around waiting, claiming, planning, and execution phases.
- [ ] Approved-review tasks are discovered by reviewer supervision and handed to merge flow without hidden channels.

## Implementation

- `internal/agent/supervisor.go` — role-specific supervision loop, wait/claim orchestration, and merge handling for reviewers.
- `internal/agent/claiming.go` — coder/reviewer claim selection and handoff resume behavior.
- `internal/agent/workdetection.go` — planner wake trigger detection and priority ordering.
