# Task Management

## Goal

Enforce a deterministic claim-to-verdict lifecycle so tasks progress through implementation, review, and escalation with explicit evidence and ownership.

## User Stories

- 1.3.1 (implemented) — As a coder/reviewer, I want a strict claim-to-verdict lifecycle so task ownership and transitions stay auditable.
- 1.3.2 (planned) — As a reviewer, I want integration-fix loop guidance so failed merges re-enter implementation with clear ownership.
- 1.3.3 (planned) — As a planner, I want automatic rescope prompts when iteration/review limits are exhausted.

## Success Criteria

- Claim operation validates dependencies, assignment availability, and source status before transitioning task to IMPLEMENTING.
- Submit-for-review rebases task branch onto integration and records review commit before READY_FOR_REVIEW.
- Verdict operation enforces APPROVED/REJECTED semantics, including mandatory rejection reason.
- Review-cycle and iteration limits escalate deadlocked tasks to BLOCKED with planner-facing questions.
- Task history records claim, review submission, verdict, and escalation events.
