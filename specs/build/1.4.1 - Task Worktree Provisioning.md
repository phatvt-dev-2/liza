# Task Worktree Provisioning

## User Story

As a coder, I want a dedicated task worktree so that my implementation is isolated from other in-flight tasks.

## Acceptance Criteria

- [ ] Claiming a READY task provisions a dedicated task branch/worktree from `integration`.
- [ ] Task claim records `base_commit` so drift can be evaluated later.
- [ ] Reassignment behavior is deterministic: same coder can reuse worktree; different coder gets a fresh one.
- [ ] Merge/integration failures keep recovery path explicit (`INTEGRATION_FAILED`) without corrupting shared branch state.

## Implementation

- `internal/ops/claim_task.go` — task claim flow, reassignment semantics, base commit/worktree assignment.
- `internal/git/worktree.go` — worktree create/remove/list and branch operations.
- `internal/ops/wt_merge.go` — safe integration merge path and failure transitions.
- `specs/protocols/worktree-management.md` — lifecycle and merge protocol definition.
