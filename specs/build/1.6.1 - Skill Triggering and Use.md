# Skill Triggering and Use

## User Story

As an agent, I want explicit skill triggers so that I can apply the right protocol for the task domain instead of using ad-hoc behavior.

## Acceptance Criteria

- [ ] Skill catalog is available to agents with names, descriptions, and paths.
- [ ] When a task matches a skill trigger (or user names a skill), the corresponding skill is loaded and followed.
- [ ] Mandatory domains (debugging/testing/code-review/architecture) invoke their required skills.
- [ ] Skills operate within contract constraints and cannot override gate semantics.

## Implementation

- `skills/*/SKILL.md` — domain protocols and trigger definitions.
- `contracts/CORE.md` — mandatory skill trigger clauses and contract precedence.
- `internal/embedded/embedded.go` — packaging/distribution of skill files.
- `internal/commands/setup.go` — installation of skills into runtime config.
