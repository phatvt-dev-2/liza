# Contract System

## Goal

Establish enforceable behavioral constraints so agents operate as reliable engineering peers instead of optimistic code generators.

## User Stories

- 1.1.1 (implemented) — As a pairing user, I want hard invariants and approval gates so I can trust state changes.
- 1.1.2 (planned) — As a multi-agent operator, I want role- and mode-specific gate semantics so autonomous execution remains controlled.
- 1.1.3 (planned) — As a security-conscious maintainer, I want explicit secrets and recovery protocols so failures are contained and auditable.

## Success Criteria

- Agents load CORE + mode contract before acting and respect forbidden state-machine transitions.
- Tier 0 invariants are never bypassed: no unapproved state change, no fabrication, no test corruption, no unvalidated success, no secret exposure.
- Pairing mode requires explicit human gate clearance before state-changing execution.
- Multi-agent mode requires pre-execution checkpoints and role-boundary enforcement.
- Security and recovery protocols provide explicit handling for contradictions, tool failures, and reset scenarios.
