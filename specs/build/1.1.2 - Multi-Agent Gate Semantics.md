# Multi-Agent Gate Semantics

## User Story

As a multi-agent operator, I want role- and mode-specific gate semantics so autonomous execution remains controlled.

## Acceptance Criteria

- [ ] Gate artifacts are explicit and mode-dependent (pairing approval, multi-agent checkpoint, subagent internal intent gate).
- [ ] Execution transitions cannot bypass required gate states.
- [ ] Role-specific workflows respect gate boundaries during autonomous loops.
- [ ] Gate semantics are externally observable through state and logs.
