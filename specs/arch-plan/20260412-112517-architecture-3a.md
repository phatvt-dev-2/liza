# Architecture Plan: MCP Server Removal and Bootstrap Replacement

**Task:** architecture-3a
**Date:** 2026-04-12
**Goal spec:** specs/goals/20260412-cli-native-access-control.md (Section 3)
**Scope:** Code deletion, 7 bootstrap integration point replacements, embedded config updates. Excludes prompt/documentation updates (Phase 4).

## Context

The MCP server (`liza-mcp`) and package (`internal/mcp/`) provide no unique value once
CLI RBAC (`--agent-id` validation) and `--json` structured output are in place (Phases 1-2).
This plan covers the safe removal of all MCP code and the replacement of 7 bootstrap
integration points that currently depend on `liza-mcp`.

### Current MCP Footprint

| Area | Files | Import graph |
|------|-------|-------------|
| Binary entrypoint | `cmd/liza-mcp/main.go` | imports `internal/mcp`, `internal/embedded` |
| Server package | `internal/mcp/` (18 .go files, 2 subdirs) | imports `internal/ops`, `internal/models`, `internal/mcp/protocol` |
| Protocol sub-package | `internal/mcp/protocol/` | no external dependents |
| Embedded config | `internal/embedded/mcp.json` | embedded into `internal/embedded/embedded.go` |

**Key finding:** Only `cmd/liza-mcp/main.go` imports `internal/mcp`. No other production code depends on the MCP package. Deletion is safe once build targets and integration references are removed.

### MCP Resources

Three read-only resources (`liza://state`, `liza://tasks`, `liza://agents`) have direct CLI equivalents (`liza get tasks --json`, `liza get agents --json`, `liza status --json`). They are removed with the `internal/mcp/` package — no separate replacement needed.

## 7 Bootstrap Integration Points

Each integration point is documented with current behavior, replacement action, and the specific file location.

### IP-1: `internal/embedded/mcp.json`

**Current:** Embedded MCP config template defining the `liza` MCP server entry. Loaded via `//go:embed "mcp.json"` in `internal/embedded/embedded.go:37-38` as `mcpSettingsContent`.

**Replacement:**
- Delete `internal/embedded/mcp.json`
- Remove `//go:embed "mcp.json"` directive and `var mcpSettingsContent []byte` from `embedded.go`
- Remove `WriteMCPSettings()` function (~45 lines, `embedded.go:529-573`)
- Remove `mergeMCPSettings()` function (~25 lines, `embedded.go:662-689`)

### IP-2: `internal/commands/init.go:560-564`

**Current:** `liza init` calls `embedded.WriteMCPSettings()` to write/merge `.mcp.json` to the project root.

**Replacement:**
- Remove the `WriteMCPSettings` call block (lines 560-565: comment + call + error handling)
- No replacement needed — agents use `Bash(liza ...)` directly, no `.mcp.json` required

### IP-3: `internal/ops/wt_create.go:128`

**Current:** `ProvisionClaudeConfig()` copies `.mcp.json` from project root into worktrees so MCP is available to agents.

**Replacement:**
- Remove `".mcp.json"` from the `files` slice at line 128
- Comment on line 119-121 mentioning "MCP access" should be updated to reflect CLI-only access

### IP-4: `internal/agent/supervisor.go:371-391`

**Current:** `buildCodexArgs()` constructs `-c mcp_servers.liza.command=...` and `-c mcp_servers.liza.args=[...]` arguments for Codex agent launch.

**Replacement:**
- Remove MCP server config args (lines 373-374)
- Remove associated comment about `destructiveHint` (lines 381-386) — no longer relevant when MCP is gone
- Keep: `exec` mode args, `--full-auto`, `--json`

### IP-5: `internal/embedded/claude-settings.json:35-36`

**Current:** Two lines:
```json
"enableAllProjectMcpServers": true,
"enabledMcpjsonServers": ["liza"],
```

**Replacement:**
- Remove `"enabledMcpjsonServers": ["liza"]` line entirely (no Liza MCP server to enable)
- Keep `"enableAllProjectMcpServers": true` — user may have other MCP servers (filesystem, JetBrains, etc.)

### IP-6: `internal/embedded/claude-settings.json:44-69`

**Current:** 26 `mcp__liza__*` permission entries allowing Claude Code to call Liza MCP tools without prompting.

**Replacement:**
- Remove all 26 `mcp__liza__*` entries (lines 44-69)
- `Bash(liza:*)` at line 97 already permits all `liza` CLI invocations — no new permission needed

### IP-7: `contracts/contract-activation.md`

**Current:** Documents `mcp_servers.liza` configuration for Codex, including Vibe/Mistral MCP setup.

**Replacement:** Phase 4 scope (prompt and documentation updates). Noted here for completeness but NOT included in downstream code-planning tasks.

## Safe Deletion Order

The deletion must avoid intermediate build failures. The dependency chain:

```
Task 0 (integration points)  ─┐
                               ├──> Task 2 (delete MCP packages)
Task 1 (build/release)       ─┘
```

1. **Task 0 — Integration points:** Remove all code references to MCP from non-MCP source files. Build and tests can still pass because `cmd/liza-mcp/` and `internal/mcp/` compile independently (no external dependents).

2. **Task 1 — Build tooling:** Remove `liza-mcp` from `Makefile`, `.goreleaser.yaml`, `install.sh`, `.gitignore`. Can execute in parallel with Task 0 since files don't overlap. After this, `make build` no longer attempts to build `liza-mcp`.

3. **Task 2 — Package deletion:** Delete `cmd/liza-mcp/`, `internal/mcp/`, and the now-obsolete lesson file `lessons/agents/cli-mcp-surface-sync.md`. Depends on both Task 0 and Task 1 completing first.
