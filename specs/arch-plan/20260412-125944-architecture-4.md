**Task:** architecture-4

**Depends on:** architecture-3 (MCP server removal and bootstrap replacement)

## Context

After Phases 1-3 complete (CLI RBAC, `--json` output, MCP server deletion), the MCP server no longer exists. But agent prompts, contracts, documentation, and lessons still reference it. This phase removes all stale Liza MCP references from non-code artifacts and rewrites agent prompt instructions to use CLI-only patterns.

### Key Distinction: Liza MCP vs External MCP

Only **Liza MCP** (`liza-mcp`, `mcp__liza__*`, `.mcp.json` Liza server entry) references are removed. External MCP servers (JetBrains, filesystem, morph-mcp, perplexity, context7, Ref, deepwiki, fetch) are unrelated and remain untouched.

### ToolPrefix and the Prompt Pipeline

The prompt system uses `MCPToolPrefix(cliName)` to inject `mcp__liza__` (Claude) or `""` (Codex) before all Liza tool names in templates. This prefix is threaded through:

1. `internal/prompts/role_context.go` defines `MCPToolPrefix()` and `toolSearchHint()`
2. `internal/prompts/templates.go` registers `toolSearchHint` as a template function (line 26)
3. `internal/agent/prompt.go` calls `MCPToolPrefix()` 3 times, stores in `ToolPrefix` fields
4. `RoleContextData.ToolPrefix` and `BasePromptConfig.ToolPrefix` carry the prefix into templates
5. Templates use `{{.ToolPrefix}}liza_get`, `{{.ToolPrefix}}liza_submit_for_review`, etc.
6. `toolSearchHint()` builds `ToolSearch select:mcp__liza__liza_get,...` hints

After MCP removal, all Liza operations are CLI commands via Bash. The `ToolPrefix` mechanism and MCP tool call syntax must be replaced with CLI command invocations.

## Prompt Template Rewrite Strategy

### Before (MCP pattern)
```
- {{.ToolPrefix}}liza_submit_for_review — Submit completed work for review
  {"task_id": "{{.TaskID}}", "commit_sha": "<SHA>", "agent_id": "{{.AgentID}}"}
```

### After (CLI pattern)
```
- liza submit-for-review {{.TaskID}} <SHA> --agent-id {{.AgentID}} --json
  (Get SHA first: git -C {{.Worktree}} rev-parse HEAD)
```

### Transformation Rules

1. **Tool prefix removal:** Delete `{{.ToolPrefix}}` from all 20 template files. Remove `ToolPrefix` field from `RoleContextData`, `BasePromptConfig`, `wakeTemplateData`, `wakePlanningCompleteData`. Remove `toolSearchHint` template function registration from `templates.go`.

2. **MCP tool call → CLI command:** Convert `{"param": "value"}` JSON syntax to CLI flag syntax. Map tool names: underscore to hyphen (`liza_submit_for_review` → `liza submit-for-review`).

3. **ToolSearch hints removal:** Delete all `toolSearchHint` calls from templates and Go code. Delete the `toolSearchHint()` function from `role_context.go`. Remove the template function registration from `templates.go`. CLI commands don't need deferred tool resolution.

4. **`--json` flag:** Add `--json` to all CLI commands in templates. Agents parse structured JSON output instead of MCP response objects.

5. **MCP fallback removal:** Delete the "If the MCP server itself is unavailable, fall back to the CLI" block from `base_prompt.tmpl`. Delete "All liza_* operations are MCP tool calls" and "MCP TOOL DISCOVERY" instructions. CLI is now the only path.

6. **"MCP tool" language → "CLI command":** Replace "use MCP tools", "execute MCP tools", "invoke MCP tools" with "use CLI commands" or "run CLI commands". Replace "MCP tool call" with "CLI command" in verdict and submission templates.

7. **Orchestrator inline tools (`builder.go`):** The orchestrator tool section (lines 120-166) uses `fmt.Sprintf` with `toolPrefix` to render MCP-style tool calls. Rewrite to emit CLI commands directly, removing the `toolPrefix` parameter from `RenderOrchestratorDashboard`.

8. **Reviewer verdict emphasis:** Current emphasis is "this is an MCP tool, NOT a CLI command" (preventing surface confusion). After removal, change to "you MUST execute this CLI command in this session" — same intent, simpler framing.
