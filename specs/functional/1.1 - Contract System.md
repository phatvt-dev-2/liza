# Contract System

Behavioral contracts governing agent conduct across pairing, multi-agent, and subagent execution.

## Description

The contract system defines the operating constraints for Liza agents: hard invariants, readiness gates, validation requirements, and reset behavior. It is enforced through contract files loaded at session bootstrap, then specialized by mode.

Contract behavior is split between universal rules (CORE), mode contracts (PAIRING, MULTI_AGENT, SUBAGENT), and the tool sub-contract. Together they define how agents decide, act, validate, and recover.

## Features

- 1.1.1 — [Tier System](../../contracts/CORE.md) — Priority hierarchy (Tier 0-3) with explicit degradation policy
- 1.1.2 — [Golden Rules](../../contracts/CORE.md) — Integrity, scope discipline, root-cause, and anti-gaming constraints
- 1.1.3 — [Execution State Machine](../../contracts/CORE.md) — Required transitions, gate artifacts, and forbidden transitions
- 1.1.4 — [Pairing Mode](../../contracts/PAIRING_MODE.md) — Human-supervised execution with approval-gated state changes
- 1.1.5 — [Multi-Agent Mode](../../contracts/MULTI_AGENT_MODE.md) — Peer-supervised execution with pre-execution checkpoints
- 1.1.6 — [Subagent Mode](../../contracts/SUBAGENT_MODE.md) — Delegated read-only/subtask execution model
- 1.1.7 — [Tool Sub-Contract](../../contracts/AGENT_TOOLS.md) — MCP-first tool selection and operation-specific defaults
- 1.1.8 — [Security + Recovery](../../contracts/CORE.md) — Secrets handling, destructive-op gates, RESET/conflict/tool-failure protocols

## Boundaries

**In scope:**
- Agent behavioral constraints and gate semantics
- Mode-specific operating contracts
- Tool usage policy for agents
- Security and recovery procedures

**Out of scope:**
- Runtime coordination mechanics (see 1.2 Multi-Agent Coordination)
- Task lifecycle and review loop (see 1.3 Task Management)
- Domain methodology content inside skills (see 1.6 Skills)
