   ```go
   func ensureRejectedWorktreeExists(
       gitWrapper *git.Git,
       ctx *claimContext,
   ) (claimWorktreePhaseResult, error) {
       result := claimWorktreePhaseResult{}
       branchName := paths.TaskBranchPrefix + ctx.taskID

       worktreeDirExists := false
       if _, err := os.Stat(ctx.worktreeDir); err == nil {
           worktreeDirExists = true
       } else if !os.IsNotExist(err) {
           return result, fmt.Errorf("failed to stat worktree %s: %w", ctx.worktreeDir, err)
       }

       branchExists := false
       if worktreeDirExists {
           var err error
           branchExists, err = gitWrapper.BranchExists(branchName)
           if err != nil {
               return result, fmt.Errorf("failed to check branch %s: %w", branchName, err)
           }
       }

       if worktreeDirExists && branchExists {
           // Both worktree and branch exist — validate and preserve
           if err := validateRejectedWorktree(gitWrapper, ctx.taskID, ctx.worktreeDir, ctx.worktreeRel); err != nil {
               return result, err
           }
           return result, nil
       }

       // Either worktree or branch (or both) missing — clean partial state and recreate

       if worktreeDirExists {
           // Worktree dir exists but branch is missing — remove orphaned worktree dir
           if err := gitWrapper.RemoveWorktree(ctx.taskID); err != nil {
               return result, fmt.Errorf("failed to remove orphaned worktree %s (branch missing): %w", ctx.worktreeRel, err)
           }
       }

       // Cleanup stale branch if any (branch without worktree, or leftover after RemoveWorktree)
       _ = gitWrapper.DeleteBranch(branchName)

       if _, err := gitWrapper.CreateWorktree(ctx.taskID, ctx.integrationBranch); err != nil {
           return result, fmt.Errorf("failed to recreate worktree for REJECTED task from integration branch: %w", err)
       }
       result.created = true
       return result, nil
   }
   ```

   **Key difference from iteration 3:** The function now explicitly checks `BranchExists` when the worktree directory exists. Three cases:
   - **dir + branch both present** → validate and preserve (existing work kept)
   - **dir present, branch missing** → remove orphaned worktree directory, then recreate from integration
   - **dir missing** → cleanup stale branch if any, recreate from integration

   This matches the spec requirement (Change 7): "preserve only when both worktree and branch exist and recreate when either is missing."

## Rejection Feedback Resolution

### Blocker 1 (iteration 3 → 4): TransitionToNewAttempt failure propagation in SubmitVerdict

**Problem:** Task 2 (iteration 3) downgraded `TransitionToNewAttempt()` failure to a warning and still returned a `VerdictResult` with `NewAttemptTriggered = false`. This violates the goal spec's postcondition: the `LimitActionNewAttempt` path must return with the task in its initial status. On failure, the task would be stuck in REJECTED with exhausted caps — unrecoverable through normal flow.

**Resolution:** Task 2 step 5 now propagates the error: `SubmitVerdict` returns `(nil, error)` when `TransitionToNewAttempt` fails. The error message includes "rejection committed" to indicate that the bb.Modify side-effects (review cycle increment, rejection reason) are persisted. The `VerdictResult` is only returned when the full operation succeeds — either a normal rejection (no cap hit), a successful new-attempt transition, or a successful BLOCKED escalation. The done_when item (c) is updated to specify "failure propagated as error from SubmitVerdict (not downgraded to warning)."

### Blocker 2 (iteration 3 → 4): Branch-missing recovery in ensureRejectedWorktreeExists

**Problem:** Task 4 (iteration 3) only branched on worktree directory existence. When the directory exists but the task branch is missing, `validateRejectedWorktree` would error with an unrecoverable failure. The spec (Change 7) requires: preserve only when BOTH worktree and branch exist; recreate when EITHER is missing.

**Resolution:** Task 4 step 4 now implements three-way branching:
1. **dir + branch both present** → validate and preserve (unchanged from iteration 3)
2. **dir present, branch missing** → `RemoveWorktree` to clean orphaned dir, then recreate from integration (NEW)
3. **dir missing** → cleanup stale branch, recreate from integration (unchanged from iteration 3)

Added test `TestClaimTask_RejectedWorktreeDirExistsBranchMissing_Recreated` to cover case 2. The compliance matrix row 13 is updated to "three-way: dir+branch → preserve; dir only → clean and recreate; neither → recreate." The done_when item (c) is updated to enumerate all three cases.

### Prior feedback status (iterations 1-3):
- RESOLVED (still resolved): The dangling CP4 ownership claim for `internal/commands/submit_verdict.go` was removed in iteration 3. The display follow-up remains explicitly marked unplanned/out of scope.
- RESOLVED (still resolved): Task 2 does not include `internal/commands/submit_verdict.go`. SubmitVerdict work is in `internal/ops/submit_verdict.go` and tests only.
