## Problem

When the `Attempted []string` field (yaml tag: `attempted`) was replaced by `Attempt int` (yaml tag: `attempt`) in commit 958180d, no migration path was added. Legacy state files containing `attempted:` load that key into the `Extra map[string]any` (the `,inline` catch-all on Task, line 207) instead of populating `Attempt`. `EffectiveAttempt()` then returns 1 for these tasks, silently granting fresh retries and suppressing "final attempt" behavior.

## Design

Three complementary layers, each addressing a different execution path:

1. **Model fallback** (task.go): `EffectiveAttempt()` checks `Extra["attempted"]` when `Attempt` is unset. Defensive â€” works even if normalization didn't run.
2. **Read-normalization** (blackboard.go): A `normalizeTaskAttempts()` function (modeled after existing `normalizeAgentRoles()`) converts legacy fields in-memory on every read. Downstream code sees a clean `Attempt` field.
3. **Persistent migration** (migrate.go): `MigrateCommand` converts `attempted:` â†’ `attempt:` on disk, eliminating the legacy field permanently.

### Conversion logic

The old formula was `len(task.Attempted) + 1` (per prompt.go before 958180d). A shared helper `MigrateAttemptedField()` on Task implements:

```
if Attempt > 0: already migrated, return false
if Extra["attempted"] exists:
  - type-assert as []any
  - if len > 0: set Attempt = min(len + 1, 2)
  - else: leave Attempt at 0 (EffectiveAttempt returns 1)
  - delete Extra["attempted"]
  - return true (changed)
return false
```

The cap at 2 reflects the hard-coded max attempts (Design Decision Â§1 of 20260323-fix-attempt-model-drift.md).

### Why three layers

| Layer | Protects against | When it runs |
|-------|------------------|--------------|
| EffectiveAttempt fallback | Code paths constructing Tasks outside blackboard (tests, raw unmarshal) | On every EffectiveAttempt() call |
| Read-normalization | Normal blackboard reads of un-migrated state files | On every Read/ReadCached/Modify |
| Persistent migration | Repeated normalization overhead; legacy field accumulation | Once, via `liza migrate` |
