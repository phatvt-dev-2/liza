**Intent:** Prevent a second reviewer from claiming a task while another reviewer holds `ReviewingBy` ownership (set during `await_resubmission`).

**Files:**

```go
case reviewerRole:
    if t.ReviewCommit == nil {
        return false
    }
    if t.ReviewingBy != nil {
```

This is defense-in-depth: existing flows are unaffected because `ReviewingBy` is always nil for claimable tasks in the current system. The guard becomes active once `await_resubmission` sets `ReviewingBy` on non-reviewing-state tasks.

## Task 3: Extend ClearStaleReviewClaims for crash recovery

**Intent:** Clear orphaned `ReviewingBy` on non-reviewing tasks when the review lease has expired, preventing permanently blocked reviewer claims after a crash during `await_resubmission`.

**Spec ref:** Spec step 2, "ClearStaleReviewClaims — Crash recovery" section.

**Files:**
- `internal/ops/clear_stale_review_claims.go` — Extend the task loop

**Changes:**
After the existing `detectReviewingState` check (which handles REVIEWING/REVIEWING_2 tasks), add a second block that handles the case where `match == nil` (task is NOT in a reviewing state) but `task.ReviewingBy != nil`. In this case:
- Check if `task.ReviewLeaseExpires` is nil or expired.
- If expired/missing: clear `task.ReviewingBy` and `task.ReviewLeaseExpires`.
- Release the reviewer agent if still assigned to this task via `state.ReleaseAgent`.
- Log the cleanup.
- Increment `cleared` counter.

Critically, this does NOT change task status (unlike the existing reviewing-state path which transitions REVIEWING -> SUBMITTED). The task stays in its current status (REJECTED, IMPLEMENTING, SUBMITTED, etc.) — only the orphaned `ReviewingBy` ownership is cleared.

```go
// After the existing reviewing-state scan:
if match == nil && task.ReviewingBy != nil {
    // Not in a reviewing state, but ReviewingBy is set — orphaned from await_resubmission crash.
    var expiredAt string
    switch {
    case task.ReviewLeaseExpires == nil:
        expiredAt = "unknown (lease missing)"
    case !task.ReviewLeaseExpires.After(now):
        expiredAt = task.ReviewLeaseExpires.Format(time.RFC3339)
    default:
        continue // lease still valid
    }

    staleReviewer := *task.ReviewingBy
    task.ReviewingBy = nil
    task.ReviewLeaseExpires = nil

    if a, ok := state.Agents[staleReviewer]; ok {
        if a.CurrentTask != nil && *a.CurrentTask == task.ID {
            state.ReleaseAgent(staleReviewer)
        }
    }

    // Log entry (same pattern as existing path)
    detail := fmt.Sprintf("Orphaned ReviewingBy cleared (expired at %s, reviewer: %s, task status: %s)",
        expiredAt, staleReviewer, task.Status)
    logEntry := log.Entry{...}  // same structure as existing
    logger.Append(logEntry)
    cleared++
}
```

**Done when:** `ClearStaleReviewClaims` clears orphaned `ReviewingBy` on non-reviewing tasks with expired leases, without changing task status. `go test ./internal/ops/... -run TestClearStaleReview` passes (all existing subtests unaffected). `go vet ./internal/ops/...` passes.

## Task 5: Implement AwaitResubmission core blocking logic

**Intent:** Create the `AwaitResubmission` operation that blocks a reviewer session after rejection until the doer resubmits, enabling same-session re-review.

**Spec ref:** Spec step 3, "internal/ops/await_resubmission.go — Core blocking logic" section.

**Files:**
- `internal/ops/await_resubmission.go` — Create new file

**Changes:**
Create `internal/ops/await_resubmission.go` with the following exported API:

Constants:
```go
const (
    ResubmissionResubmitted = "RESUBMITTED"
    ResubmissionTerminal    = "TERMINAL"
    ResubmissionTimeout     = "TIMEOUT"
    ResubmissionAborted     = "ABORTED"
)
```

Result type:
```go
type AwaitResubmissionResult struct {
    Verdict      string
    TaskStatus   models.TaskStatus
    Reason       string
    ReviewCommit string
    ReviewCycle  int
}
```

Main function:
```go
func AwaitResubmission(ctx context.Context, projectRoot, taskID, agentID string, timeout time.Duration) (*AwaitResubmissionResult, error)
```

Internal structure follows `AwaitVerdict` patterns:

1. **Precondition validation:**
   - Task exists, has role_pair
   - Task status is rejected OR already submitted (via `resolver.RejectedStatus` / `resolver.SubmittedStatus`). Accepting submitted handles the fast-doer edge case.
   - Agent was the last rejecting reviewer: scan `task.History` in reverse for most recent `TaskEventReviewVerdictRejected` by this agent.

2. **Acquire review ownership** — atomically via `bb.Modify`:
   - `task.ReviewingBy = &agentID`
   - `task.ReviewLeaseExpires = now + timeout + 5min`
   - `agent.Status = AgentStatusWaiting`
   - `agent.CurrentTask = &taskID`

3. **Early resubmission** — if task is already submitted at entry, skip wait loop and jump to reclaim.

4. **Watch loop** — same pattern as `AwaitVerdict`:
   - `watcher.Events()` -> check status via `checkResubmissionStatus()`
   - `abortTicker` (1s) -> check system mode -> ABORTED
   - `deadlineTimer` -> TIMEOUT
   - `ctx.Done()` -> context error
   - `watcher.Errors()` -> polling fallback

5. **checkResubmissionStatus()** — detect when task exits the waiting set:
   - `submitted` -> RESUBMITTED (resubmission detected)
   - `approved`, `blocked`, `superseded`, cancelled, `integration_failed` -> TERMINAL
   - All other statuses -> keep waiting

6. **Result mapping — RESUBMITTED** — atomically via `bb.Modify`:
   - `task.TransitionWith` from SUBMITTED -> REVIEWING
   - Fresh `ReviewLeaseExpires = now + 30min`
   - `agent.Status = AgentStatusReviewing`
   - `agent.CurrentTask = &taskID`
   - Return with `ReviewCommit` and `ReviewCyclesCurrent`

7. **Result mapping — terminal/timeout/abort** — release via `releaseReviewOwnership`:
   - Clear `task.ReviewingBy`, `task.ReviewLeaseExpires`
   - Clear `agent.CurrentTask`
   - Agent status intentionally left unchanged (same as `releaseOwnership` in await_verdict)

8. **releaseReviewOwnership()** — helper to clear both agent and task fields.

9. **Polling fallback** — `awaitResubmissionPolling()` following the same pattern as `awaitVerdictPolling`.

Reuses: `readTaskState`, `loadResolver`/`loadPipelineBundle`, `bb.WatchForChanges()`, `PreconditionError`/`OperationalError` types, `BuildPipelineTransitions`.

**Done when:** `internal/ops/await_resubmission.go` compiles (`go build ./internal/ops/...`), exports `AwaitResubmission`, `AwaitResubmissionResult`, and the four `Resubmission*` constants. `go vet ./internal/ops/...` passes. The function validates preconditions (task exists, rejected/submitted status, agent was last rejecting reviewer), acquires review ownership atomically, watches for resubmission or terminal states, and returns the appropriate verdict with ownership correctly managed.
