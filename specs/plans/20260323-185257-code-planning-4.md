## Rejection Feedback Resolution

### Blocker 1 (iteration 2 ŌåÆ 3): Dangling cross-phase ownership claim for TransitionToNewAttempt

**Problem:** The iteration-2 plan header and Cross-Phase Boundary Notes stated CP3 "provides `TransitionToNewAttempt` (which appends the `new_attempt` history entry with reason)". But `TransitionToNewAttempt` and its `new_attempt` history append are owned by CP2 Task 4, not CP3. CP3 only provides the trigger-path wiring that *calls* `TransitionToNewAttempt`.

**Resolution:** The dependency header (line 7) now correctly attributes `TransitionToNewAttempt` ownership to CP2 Task 4 and describes CP3's contribution as "trigger-path wiring that calls `TransitionToNewAttempt` from `SubmitVerdict` and `ClaimTask`". The Cross-Phase Boundary Notes section explicitly separates CP2 (owns the operation and history entry) from CP3 (wires the call sites). All references to `TransitionToNewAttempt` throughout the plan cite CP2 Task 4 as the source.

### Blocker 2 (iteration 2 ŌåÆ 3): Spec Compliance Matrix rows with unverified done_when coverage

**Problem:** Three matrix rows overstated coverage:
- Row 2: "prior rejection gate `task.Iteration > 1` unchanged" marked Covered for Task 2, but Task 2's done_when never asserted the gate remains unchanged.
- Rows 9 and 11: Change 13 bullet 3 / Design Decision #4 "doer prompts only" marked Covered for Task 5, but Task 5's done_when only verified positive inclusion (coder and code-planner contain prior-attempt), never falsified reviewer-role leakage.

**Resolution:**
- **Row 2**: Task 2 now has a 3rd test (`TestBuildTaskRoleContextData_PriorRejectionGate_Attempt2Iteration1_Empty`) and done_when clause (e) that explicitly verifies the `task.Iteration > 1` gate is unchanged: at attempt 2, iteration 1, with non-empty `RejectionReason`, `PriorRejection` is empty.
- **Rows 9 and 11**: Task 5 now has a 3rd test (`TestBuildPrompt_PriorAttemptOutcome_ReviewerExcluded`) and done_when clause (e) that falsifies reviewer-role leakage: code-reviewer prompt at attempt 2 does NOT contain "PRIOR ATTEMPT OUTCOME".

### Prior iteration resolutions (still resolved):
