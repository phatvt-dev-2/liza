## Task 1: Replace checkReassigned with attempt-aware messaging

**Files:**
- `internal/commands/watch.go` — rewrite `checkReassigned()` function
- `internal/commands/watch_test.go` — replace existing `TestCheckReassigned` test cases

**Changes:**

1. **`internal/commands/watch.go`** — Replace `checkReassigned()` (lines 430-465). The current implementation warns on identity-based reassignment (current assignee != first claimer) and filters for IMPLEMENTING status only. Replace with attempt-aware logic:
   - Iterate all tasks (no status filter — attempt 2 is meaningful regardless of current status; a task may be in REJECTED, IMPLEMENTING, or any other non-terminal status at attempt 2).
   - Check `task.EffectiveAttempt() == 2` (uses CP2's `EffectiveAttempt()` method).
   - Use cache key `"attempt2:"+task.ID` to alert once per task.
   - Alert level: warning. Category: `"ATTEMPT"`. Message: `fmt.Sprintf("%s — attempt 2 (final attempt)", task.ID)`.
   - Remove all identity-based reassignment logic (first-claimer history scan, assignee comparison).

2. **`internal/commands/watch_test.go`** — Replace existing `TestCheckReassigned` test cases with attempt-aware tests:
   - `"attempt 2 IMPLEMENTING task alerts"`: Task with `Attempt: 2`, IMPLEMENTING status. Empty cache → 1 alert with Category `"ATTEMPT"` and message containing `"attempt 2 (final attempt)"`.
   - `"attempt 2 REJECTED task alerts"`: Task with `Attempt: 2`, REJECTED status (with `RejectionReason` set to satisfy validator). Empty cache → 1 alert with Category `"ATTEMPT"` and message containing `"attempt 2 (final attempt)"`. This test falsifies the old IMPLEMENTING-only filter.
   - `"attempt 1 task no alert"`: Task with `Attempt: 1`. Empty cache → 0 alerts.
   - `"attempt 0 (legacy) no alert"`: Task with `Attempt: 0` (unset/legacy). Empty cache → 0 alerts. `EffectiveAttempt()` returns 1 for `Attempt: 0`, so no alert.
   - `"attempt 2 cached suppresses duplicate"`: Task with `Attempt: 2`. Cache with `"attempt2:task-1"` entry → 0 alerts (deduplicated).

**done_when:** (a) `checkReassigned()` in `internal/commands/watch.go` iterates all tasks regardless of status (no IMPLEMENTING filter), checks `task.EffectiveAttempt() == 2`, and alerts with category `"ATTEMPT"` and message `"<taskID> — attempt 2 (final attempt)"`. No identity-based assignee comparison remains. (b) `TestCheckReassigned` in `internal/commands/watch_test.go` has 5 subtests: "attempt 2 IMPLEMENTING task alerts" passes asserting 1 alert with correct category/message, "attempt 2 REJECTED task alerts" passes asserting 1 alert (proving no status filter), "attempt 1 task no alert" passes asserting 0 alerts, "attempt 0 (legacy) no alert" passes asserting 0 alerts, "attempt 2 cached suppresses duplicate" passes asserting 0 alerts.

**scope:** `internal/commands/watch.go`, `internal/commands/watch_test.go`.

## Task 4: Add checkStaleSentinels function

**Files:**
- `internal/commands/watch.go` — add `StaleSentinelThreshold` constant, `checkStaleSentinels()` function, wire into `runChecks()`
- `internal/commands/watch_test.go` — add `TestCheckStaleSentinels` test

**Changes:**

1. **`internal/commands/watch.go`** — Add constant after existing constants (line 31):
   ```go
   StaleSentinelThreshold = 2 * time.Minute
   ```

2. **`internal/commands/watch.go`** — Add new function `checkStaleSentinels(state *models.State, cache map[string]time.Time) []alert`:
   ```go
   func checkStaleSentinels(state *models.State, cache map[string]time.Time) []alert {
       var alerts []alert
       now := time.Now().UTC()

       activeSentinels := make(map[string]bool)

       for _, task := range state.Tasks {
           if task.AssignedTo == nil || !strings.HasPrefix(*task.AssignedTo, "$") {
               continue
           }
           activeSentinels[task.ID] = true

           cacheKey := "sentinel:" + task.ID
           firstSeen, seen := cache[cacheKey]
           if !seen {
               cache[cacheKey] = now
               continue
           }
           if now.Sub(firstSeen) > StaleSentinelThreshold {
               alerts = append(alerts, alert{
                   Timestamp: now,
                   Level:     alertLevelCritical,
                   Category:  "STALE SENTINEL",
                   Message:   fmt.Sprintf("%s stuck in transition — manual repair needed", task.ID),
               })
           }
       }

       // Clear cache entries for sentinels that resolved.
       for key := range cache {
           if !strings.HasPrefix(key, "sentinel:") {
               continue
           }
           taskID := strings.TrimPrefix(key, "sentinel:")
           if !activeSentinels[taskID] {
               delete(cache, key)
           }
       }

       return alerts
   }
   ```
   Key design points:
   - Scans all tasks for `AssignedTo` starting with `$`.
   - Uses cache to track first-seen time. Alert (critical) when present for >2 minutes.
   - Clears cache when sentinel disappears (transition completed successfully).
   - Alert level is critical (not warning) — stale sentinels block the task permanently since `$transitioning` is not a registered agent and `recover_agent.go` won't catch it.
   - **Repeated alerts are intentional:** Once the threshold is crossed, the alert fires on every poll while the sentinel remains stuck. Stuck sentinels require manual repair and represent an emergency state — suppressing repeats would hide an ongoing issue that needs human attention. This is consistent with `checkReviewLoops` and `checkIntegrationFailures` which also alert every poll without suppression.

3. **`internal/commands/watch.go`** — Wire into `runChecks()` by adding to the checks slice (after `checkApproachingLimits`, before `checkStalled`):
   ```go
   func() []alert { return checkStaleSentinels(state, config.StateCache) },
   ```

4. **`internal/commands/watch_test.go`** — Add `TestCheckStaleSentinels` with 4 subtests:
   - `"detects stale sentinel"`: Task with `AssignedTo: "$transitioning"`. Cache with `"sentinel:task-1"` entry from 3 minutes ago. Assert 1 alert with Level `alertLevelCritical`, Category `"STALE SENTINEL"`, message containing `"stuck in transition"`.
   - `"no alert within threshold"`: Task with `AssignedTo: "$transitioning"`. Cache with `"sentinel:task-1"` entry from 1 minute ago. Assert 0 alerts.
   - `"first seen starts tracking"`: Task with `AssignedTo: "$transitioning"`. Empty cache. Assert 0 alerts. Assert cache now contains `"sentinel:task-1"` entry.
   - `"clears cache when sentinel resolves"`: No tasks with sentinel `AssignedTo`. Cache with stale `"sentinel:task-1"` entry. Assert 0 alerts. Assert cache no longer contains `"sentinel:task-1"`.

**done_when:** (a) `StaleSentinelThreshold` constant is `2 * time.Minute` in `internal/commands/watch.go`. (b) `checkStaleSentinels()` function exists in `internal/commands/watch.go` and scans for `$`-prefixed `AssignedTo`, uses cache tracking, and alerts at critical level with category `"STALE SENTINEL"` when sentinel age exceeds threshold. Repeated alerts on every poll while sentinel remains stuck are intentional (no suppression after threshold). (c) `checkStaleSentinels` is wired into `runChecks()` checks slice. (d) `TestCheckStaleSentinels` "detects stale sentinel" subtest passes asserting 1 critical alert with message containing `"stuck in transition"`. (e) `TestCheckStaleSentinels` "no alert within threshold" subtest passes asserting 0 alerts. (f) `TestCheckStaleSentinels` "first seen starts tracking" subtest passes asserting 0 alerts and cache populated. (g) `TestCheckStaleSentinels` "clears cache when sentinel resolves" subtest passes asserting 0 alerts and cache cleared.
