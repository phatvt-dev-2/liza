## Task 2: Update prompt.go ‚Äî AttemptNum and PriorAttemptOutcome population

**Files:**
- `internal/agent/prompt.go` ‚Äî fix AttemptNum computation, add PriorAttemptOutcome extraction
- `internal/agent/prompt_test.go` ‚Äî add 2 tests

**Changes:**

1. **`internal/agent/prompt.go`** ‚Äî Line 132: Replace `AttemptNum: len(task.Attempted) + 1` with `AttemptNum: task.EffectiveAttempt()`. The `Attempted` field is removed by CP2 Task 1 and `EffectiveAttempt()` is the new canonical source.

2. **`internal/agent/prompt.go`** ‚Äî After the prior rejection block (after line 152), add PriorAttemptOutcome extraction for doer roles when AttemptNum == 2:
   ```go
   // Prior attempt outcome (attempt 2 only)
   if data.AttemptNum == 2 {
       for i := len(task.History) - 1; i >= 0; i-- {
           if task.History[i].Event == models.TaskEventNewAttempt && task.History[i].Reason != nil {
               data.PriorAttemptOutcome = *task.History[i].Reason
               break
           }
       }
   }
   ```
   This scans history backwards for the `new_attempt` event and extracts its `Reason` field. The `TaskHistoryEntry.Reason` field (`*string`, line 85 in history.go) is populated by `TransitionToNewAttempt` (CP2 Task 4, Phase 1 step: "Append history: `TaskEventNewAttempt` with reason string"). The extraction is unconditional on roleType because the template itself gates rendering on doer-only via pipeline context-sections.

3. **`internal/agent/prompt.go`** ‚Äî Line 150: The prior rejection gate `task.Iteration > 1` is unchanged. At attempt 2 iteration 1, there's no within-attempt prior rejection (correct behavior per spec). At iteration 2+, prior rejection from within attempt 2 is shown.

4. **`internal/agent/prompt_test.go`** ‚Äî Add 2 tests that call `buildTaskRoleContextData` directly (not `testBuildPrompt`) to verify data population independently of template rendering:
   - `TestBuildTaskRoleContextData_AttemptNum_UsesEffectiveAttempt`: Create state with task having `Attempt: 2`, `Iteration: 1`. Call `buildTaskRoleContextData`. Assert `data.AttemptNum == 2`. Also test with `Attempt: 0` ‚Üí assert `data.AttemptNum == 1` (backward compat via `EffectiveAttempt()`).
   - `TestBuildTaskRoleContextData_PriorAttemptOutcome_Attempt2`: Create state with task having `Attempt: 2`, `Iteration: 1`, and a `new_attempt` history entry with `Reason: "review cycle limit reached after 5 rejections"`. Call `buildTaskRoleContextData`. Assert `data.PriorAttemptOutcome == "review cycle limit reached after 5 rejections"`. Also test with `Attempt: 1` ‚Üí assert `data.PriorAttemptOutcome == ""` (not populated for attempt 1).

**done_when:** (a) `AttemptNum` in `buildTaskRoleContextData` uses `task.EffectiveAttempt()` instead of `len(task.Attempted) + 1`. (b) When `AttemptNum == 2`, `PriorAttemptOutcome` is populated from the last `new_attempt` history entry's `Reason` field. (c) `TestBuildTaskRoleContextData_AttemptNum_UsesEffectiveAttempt` passes in `internal/agent/prompt_test.go`: data from task with `Attempt: 2` has `AttemptNum == 2`, data from task with `Attempt: 0` has `AttemptNum == 1`. (d) `TestBuildTaskRoleContextData_PriorAttemptOutcome_Attempt2` passes in `internal/agent/prompt_test.go`: data from task at attempt 2 with `new_attempt` history entry has `PriorAttemptOutcome` matching the reason; data from task at attempt 1 has empty `PriorAttemptOutcome`.

**scope:** `internal/agent/prompt.go`, `internal/agent/prompt_test.go`.

**spec_ref:** Goal spec Change 8 ‚Äî "Prompt building ‚Äî internal/agent/prompt.go"

**depends_on:** Task 1

**Files:**
- `internal/prompts/templates/blocks/assigned_task.tmpl` ‚Äî add ATTEMPT line and final-attempt warning
- `internal/prompts/templates/blocks/review_task.tmpl` ‚Äî add ATTEMPT line and final-attempt warning
- `internal/prompts/templates/blocks/prior_attempt.tmpl` (new) ‚Äî render PriorAttemptOutcome for attempt 2
- `internal/prompts/templates/blocks/doer_state_transitions.tmpl` ‚Äî update coder LIMITS line
- `internal/embedded/pipeline.yaml` ‚Äî add `prior-attempt` to doer role context-sections
- `internal/agent/prompt_test.go` ‚Äî add 3 integration tests for attempt rendering

**Changes:**

1. **`internal/prompts/templates/blocks/assigned_task.tmpl`** ‚Äî After the `ITERATION: {{.IterationNum}}` line (line 17), add:
   ```
   ATTEMPT: {{.AttemptNum}}
   {{- if eq .AttemptNum 2}}
   FINAL ATTEMPT ‚Äî task will be BLOCKED if limits are reached.
   {{- end}}
   ```

2. **`internal/prompts/templates/blocks/review_task.tmpl`** ‚Äî After the `ITERATION: {{.IterationNum}}` line (line 23), add:
   ```
   ATTEMPT: {{.AttemptNum}}
   {{- if eq .AttemptNum 2}}
   FINAL ATTEMPT ‚Äî task will be BLOCKED if limits are reached.
   {{- end}}
   ```

3. **`internal/prompts/templates/blocks/prior_attempt.tmpl`** (new file) ‚Äî Render PriorAttemptOutcome when AttemptNum == 2. Included in doer prompts only (gated by context-sections in pipeline.yaml):
   ```
   {{- define "prior-attempt" -}}
   {{- if and (eq .AttemptNum 2) (ne .PriorAttemptOutcome "")}}

   === PRIOR ATTEMPT OUTCOME ===
   {{.PriorAttemptOutcome}}
   Do NOT repeat the same approach. The prior attempt was exhausted ‚Äî try a fundamentally different strategy.
   {{- end}}
   {{- end -}}
   ```

4. **`internal/prompts/templates/blocks/doer_state_transitions.tmpl`** ‚Äî Line 10 (coder role LIMITS line): Replace:
   ```
   LIMITS: Iteration and review cycle limits are enforced ‚Äî task escalates to BLOCKED when exceeded.
   ```
   With:
   ```
   LIMITS: Iteration and review cycle limits are enforced ‚Äî task starts fresh attempt when limits reached; BLOCKED on second attempt exhaustion.
   ```

5. **`internal/embedded/pipeline.yaml`** ‚Äî Add `prior-attempt` to the `context-sections` list for every doer role that receives attempt context. Insert after `prior-rejection` in each role's section list:
   - `coder` (line 222, after `prior-rejection`): add `- prior-attempt`
   - `code-planner` (line 163, after `prior-rejection`): add `- prior-attempt`
   - `epic-planner` (line 49, after `prior-rejection`): add `- prior-attempt`
   - `us-writer` (line 128 area, after `prior-rejection`): add `- prior-attempt`

   Reviewer roles do NOT get `prior-attempt` ‚Äî the template is doer-specific (shows the coder "do not repeat the same approach" guidance).

6. **`internal/agent/prompt_test.go`** ‚Äî Add 3 tests:
   - `TestBuildPrompt_AttemptRendered_Attempt2`: Create state with task having `Attempt: 2`, `Iteration: 1`, and a `new_attempt` history entry with `Reason: "review cycle limit reached"`. Build coder prompt via `testBuildPrompt`. Assert prompt contains `"ATTEMPT: 2"`, `"FINAL ATTEMPT"`, `"PRIOR ATTEMPT OUTCOME"`, and `"review cycle limit reached"`.
   - `TestBuildPrompt_AttemptRendered_Attempt1`: Create state with task having `Attempt: 1` (or 0), `Iteration: 1`. Build coder prompt. Assert prompt contains `"ATTEMPT: 1"`. Assert prompt does NOT contain `"FINAL ATTEMPT"` or `"PRIOR ATTEMPT OUTCOME"`.
   - `TestBuildPrompt_LimitsLine_FreshAttempt`: Build coder prompt. Assert prompt contains `"task starts fresh attempt when limits reached"` (updated LIMITS line).

**done_when:** (a) `assigned_task.tmpl` renders `ATTEMPT: {{.AttemptNum}}` after ITERATION, with "FINAL ATTEMPT" warning when AttemptNum == 2. (b) `review_task.tmpl` renders same ATTEMPT line and warning. (c) `prior_attempt.tmpl` exists as a new block template that renders "PRIOR ATTEMPT OUTCOME" section when AttemptNum == 2 and PriorAttemptOutcome is non-empty. (d) `doer_state_transitions.tmpl` coder LIMITS line updated to "task starts fresh attempt when limits reached; BLOCKED on second attempt exhaustion." (e) `pipeline.yaml` has `prior-attempt` in context-sections for coder, code-planner, epic-planner, and us-writer roles (after `prior-rejection`). (f) `TestBuildPrompt_AttemptRendered_Attempt2` passes: prompt at attempt 2 contains ATTEMPT line, FINAL ATTEMPT warning, and PRIOR ATTEMPT OUTCOME with reason. (g) `TestBuildPrompt_AttemptRendered_Attempt1` passes: prompt at attempt 1 contains ATTEMPT line but not FINAL ATTEMPT or PRIOR ATTEMPT OUTCOME. (h) `TestBuildPrompt_LimitsLine_FreshAttempt` passes: prompt contains updated LIMITS text.

**scope:** `internal/prompts/templates/blocks/assigned_task.tmpl`, `internal/prompts/templates/blocks/review_task.tmpl`, `internal/prompts/templates/blocks/prior_attempt.tmpl` (new), `internal/prompts/templates/blocks/doer_state_transitions.tmpl`, `internal/embedded/pipeline.yaml`, `internal/agent/prompt_test.go`.
