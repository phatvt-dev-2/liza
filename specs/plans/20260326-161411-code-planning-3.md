# Phase 3: Data & Refresh — Code Plan

**Task:** code-planning-3
**Spec:** specs/goals/20260326-tui.md §Data Sources & Refresh, §State cache, §Activity Panel, §Anomaly Monitoring
**Scope:** `internal/tui/commands.go` (Bubbletea Cmd functions for data), data-related `internal/tui/update.go` handlers, and supporting model/type additions in `internal/tui/model.go`. ## Overview

Implement the data layer that feeds the TUI dashboard. This phase connects the Bubbletea event loop to three data sources:

1. **fsnotify** — reactive state refresh via `Blackboard.WatchForChanges()` with 50ms debounce (built into `internal/db/watcher.go`)
2. **10s poll tick** — fallback refresh, heartbeat age updates, and anomaly check execution
3. **log.yaml tailing** — incremental log reading triggered by state changes and ticks

All data flows through Bubbletea's message-passing architecture: Cmd functions run in goroutines, return typed messages, and Update handlers process them sequentially.

## Data Flow Architecture

```
                    ┌─────────────────────────────────────────────────────────┐
                    │                  Init()                                 │
                    │  tea.Batch(watchStateCmd, readStateCmd,                │
                    │            readLogCmd, tickCmd)                        │
                    └───────┬──────────┬───────────┬──────────┬─────────────┘
                            │          │           │          │
                            ▼          ▼           ▼          ▼
                    watchStateCmd  readStateCmd  readLogCmd  tickCmd
                    (blocks on     (reads         (reads     (10s timer)
                     Events())      state.yaml)   log.yaml)
                            │          │           │          │
                            ▼          ▼           ▼          ▼
                    stateChangedMsg StateMsg  LogEntriesMsg  TickMsg
                            │          │           │          │
                    ┌───────┴──────────┴───────────┴──────────┘
                    │              Update()
                    ├─ stateChangedMsg → tea.Batch(readStateCmd, readLogCmd, watchStateCmd)
                    ├─ StateMsg        → update m.state, process new anomalies, set m.ready
                    ├─ TickMsg         → tea.Batch(readStateCmd, runChecksCmd, readLogCmd, tickCmd)
                    ├─ alertsMsg       → convert to ActivityEntry, set banner if critical
                    ├─ LogEntriesMsg   → append to m.activities, update m.logPosition
                    ├─ WindowSizeMsg   → update dimensions, rebuild styles, recalc columnTier
                    ├─ errMsg          → log error (no-op for now, future: status bar)
                    └─ watcherClosedMsg → fall back to tick-only refresh
```

## Task 1: Data-layer types and model extensions (model.go)

### Intent
Add the types, interfaces, and model fields required by the data layer, and update the `New()` constructor to initialize data-layer resources (watcher, blackboard, paths).

### Design

#### StateWatcher Interface

The `db.stateWatcher` type is unexported. Define a local interface that it implicitly satisfies:

```go
// StateWatcher abstracts the fsnotify-based state watcher from internal/db.
// Satisfied by *db.stateWatcher returned from Blackboard.WatchForChanges().
type StateWatcher interface {
    Events() <-chan struct{}
    Errors() <-chan error
    Close() error
}
```

#### New Message Types

Phase 1 defined `StateMsg`, `TickMsg`, `AlertMsg`, `CmdResultMsg`, `LogEntriesMsg`. Phase 3 adds:

```go
// stateChangedMsg signals that the state file was modified (from fsnotify watcher).
// Triggers a state re-read and log re-read.
type stateChangedMsg struct{}

// watcherClosedMsg signals the fsnotify watcher channel closed unexpectedly.
// The TUI falls back to tick-only refresh.
type watcherClosedMsg struct{}

// alertsMsg carries a batch of alerts from anomaly checks.
// Includes the updated state cache to avoid data races (cache is copied
// before the Cmd goroutine runs, modified by checks, returned here).
type alertsMsg struct {
    Alerts     []AlertMsg
    StateCache map[string]time.Time
}

// errMsg carries an error from an async Cmd function.
type errMsg struct {
    err error
}

func (e errMsg) Error() string { return e.err.Error() }
```

#### LogEntriesMsg Extension

Add `NewPosition` field for incremental log tracking:

```go
type LogEntriesMsg struct {
    Entries     []log.Entry
    NewPosition int64  // byte offset after last read entry — caller updates m.logPosition
}
```

#### Additional Model Fields

```go
// Data layer (added by Phase 3)

blackboard       *db.Blackboard     // for state reads

alertsLogPath    string             // absolute path to alerts.log
lastAnomalyCount int                // tracks processed state.Anomalies for incremental sync
```

#### Updated Constructor

```go

// Signature changes from Phase 1 stub: now returns error (watcher creation can fail).
func New(projectRoot string) (Model, error)
```

The constructor:

This follows the spec's architecture: "commands.go — Bubbletea Cmd functions" and "update.go — message handling."

## Task 3: commands.go — Bubbletea Cmd Functions + Init()

**Depends on:** Task 1 (message types, Model fields), Task 2 (exported RunChecksWithState)

### Intent
Implement all data-layer Cmd functions in `internal/tui/commands.go` and the `Init()` method that bootstraps the data flow.

### Design

#### watchStateCmd — fsnotify subscription

```go
// watchStateCmd blocks on the watcher's Events channel and returns
// stateChangedMsg when the state file is modified. Returns watcherClosedMsg
// if the channel closes. Returns errMsg on watcher errors.
//
// Called from Init() and re-chained after each stateChangedMsg in Update().
func watchStateCmd(watcher StateWatcher) tea.Cmd
```

Implementation:
```go
func watchStateCmd(watcher StateWatcher) tea.Cmd {
    return func() tea.Msg {
        select {
        case _, ok := <-watcher.Events():
            if !ok {
                return watcherClosedMsg{}
            }
            return stateChangedMsg{}
        case err, ok := <-watcher.Errors():
            if !ok {
                return watcherClosedMsg{}
            }
            return errMsg{err}
        }
    }
}
```

The 50ms debounce is handled by `internal/db/watcher.go:59` (built into the watcher, not reimplemented here).

#### readStateCmd — state.yaml read

```go
// readStateCmd reads state.yaml via Blackboard.Read() and returns a StateMsg.
// Returns errMsg on read failure.
func readStateCmd(bb *db.Blackboard) tea.Cmd
```

Implementation calls `bb.Read()` and wraps the result in `StateMsg{State: state}`.

#### readLogCmd — incremental log.yaml read

```go
// readLogCmd reads new entries from log.yaml starting at the given byte offset.
// Returns LogEntriesMsg with parsed entries and the new byte position.
// Returns empty LogEntriesMsg if no new data or file doesn't exist.
func readLogCmd(logPath string, offset int64) tea.Cmd
```

Implementation:
1. `os.Open(logPath)` — return empty on `IsNotExist`
2. `f.Stat()` — if `Size() <= offset`, return empty (no new data)
3. Read bytes from `offset` to EOF via `f.ReadAt`
4. Parse bytes as YAML sequence via `yaml.Unmarshal` into `[]log.Entry`
5. Return `LogEntriesMsg{Entries: entries, NewPosition: info.Size()}`

Edge case: if partial YAML at the boundary, the unmarshal may fail. In that case, try `extractLastEntryYAML`-style boundary detection or return empty and let the next read retry (the position is not advanced on failure).

#### runChecksCmd — anomaly check execution

```go
// runChecksCmd runs all anomaly checks against the provided state snapshot.
// Copies the state cache before entering the goroutine to avoid data races.
// Writes alerts to alerts.log and returns alertsMsg with results and updated cache.
func runChecksCmd(projectRoot, alertsLogPath string, state *models.State, cache map[string]time.Time) tea.Cmd
```

Implementation:
1. **Copy cache** before returning the Cmd closure (avoid data race — the map is shared with the model):
   ```go
   cacheCopy := make(map[string]time.Time, len(cache))
   for k, v := range cache { cacheCopy[k] = v }
   ```
2. Inside the closure, call `commands.RunChecksWithState(state, config)` with the copied cache
3. Write each alert to alerts.log via `commands.WriteAlert`
4. Convert `[]commands.Alert` to `[]AlertMsg`
5. Return `alertsMsg{Alerts: alertMsgs, StateCache: cacheCopy}`

Thread safety: state is read-only in checks. Cache is copied. No shared mutable state.

#### tickCmd — 10s poll timer

```go
// tickCmd returns a tea.Cmd that fires a TickMsg after 10 seconds.
func tickCmd() tea.Cmd
```

Implementation: `tea.Tick(10*time.Second, func(t time.Time) tea.Msg { return TickMsg(t) })`

#### Init() — bootstrap

```go
// Init returns the initial Cmd batch that starts the data flow.
// Subscribes to watcher, reads initial state, reads initial log, starts tick timer.
func (m Model) Init() tea.Cmd
```

Implementation:
```go
func (m Model) Init() tea.Cmd {
    cmds := []tea.Cmd{
        readStateCmd(m.blackboard),
        readLogCmd(m.logPath, m.logPosition),
        tickCmd(),
    }
    if m.watcher != nil {
        cmds = append(cmds, watchStateCmd(m.watcher))
    }
    return tea.Batch(cmds...)
}
```

If watcher is nil (creation failed in New(), fallback mode), only tick-based refresh runs.

### Scope
- **In scope:** `internal/tui/commands.go` (new file) — watchStateCmd, readStateCmd, readLogCmd, runChecksCmd, tickCmd, Init()
- **Out of scope:** Update handlers (Task 4), key dispatch, command execution, integration

### Done When
- `internal/tui/commands.go` compiles as part of the `internal/tui` package
- `watchStateCmd(StateWatcher)` returns a `tea.Cmd` that blocks on Events() and returns `stateChangedMsg`

- `Init()` omits watchStateCmd when `m.watcher` is nil

## Task 4: update.go — Data Message Handlers

**Depends on:** Task 3 (Cmd functions to chain from handlers)

### Intent
Implement the `Update()` method in `internal/tui/update.go` with handlers for all data-layer messages and `tea.WindowSizeMsg`.

### Design

#### Update() Method

```go
// Update handles all incoming messages and returns the updated model + next Cmd.
// Phase 3 covers data messages only. Phase 4 adds key dispatch.
func (m Model) Update(msg tea.Msg) (tea.Model, tea.Cmd)
```

Uses a type switch on `msg`:

#### stateChangedMsg Handler

Triggered by fsnotify. Chains state read + log read + re-subscribe:

```go
case stateChangedMsg:
    return m, tea.Batch(
        readStateCmd(m.blackboard),
        readLogCmd(m.logPath, m.logPosition),
        watchStateCmd(m.watcher),
    )
```

#### StateMsg Handler

Updates model state, processes new blackboard anomalies:

```go
case StateMsg:
    m.state = msg.State
    m.ready = true

    // Sync new blackboard anomalies to activity feed (incremental).
    // state.Anomalies is append-only; track count for delta processing.
    if m.state != nil && len(m.state.Anomalies) > m.lastAnomalyCount {
        for _, a := range m.state.Anomalies[m.lastAnomalyCount:] {
            entry := ActivityEntry{
                Timestamp: a.Timestamp,
                Source:    "anomaly",
                Agent:     a.Reporter,
                Action:    a.Type,
                Task:      a.Task,
                Detail:    formatAnomalyDetails(a.Details),
                Level:     "⚠️",
            }
            m.activities = appendActivity(m.activities, entry)
        }
        m.lastAnomalyCount = len(m.state.Anomalies)
    }
    return m, nil
```

`formatAnomalyDetails` converts `map[string]any` to a compact string (e.g., `key1=val1 key2=val2`).

#### TickMsg Handler

10s tick fallback. Chains state read + anomaly checks + log read + next tick:

```go
case TickMsg:
    return m, tea.Batch(
        readStateCmd(m.blackboard),
        runChecksCmd(m.projectRoot, m.alertsLogPath, m.state, m.stateCache),
        readLogCmd(m.logPath, m.logPosition),
        tickCmd(),
    )
```

#### alertsMsg Handler

Processes batch alerts from anomaly checks:

```go
case alertsMsg:
    // Update state cache with modified copy from check goroutine
    m.stateCache = msg.StateCache

    for _, a := range msg.Alerts {
        entry := ActivityEntry{
            Timestamp: a.Timestamp,
            Source:    "alert",
            Action:    a.Category,
            Detail:    a.Message,
            Level:     a.Level,
        }
        m.activities = appendActivity(m.activities, entry)

        // Critical alerts (🚨) set the alert banner
        if a.Level == "🚨" {
            m.alertBanner = &entry
            m.alertExpiry = time.Now().Add(10 * time.Second)
        }
    }
    return m, nil
```

#### LogEntriesMsg Handler

Appends new log entries to the activity feed:

```go
case LogEntriesMsg:
    if msg.NewPosition > 0 {
        m.logPosition = msg.NewPosition
    }
    for _, e := range msg.Entries {
        task := ""
        if e.Task != nil {
            task = *e.Task
        }
        entry := ActivityEntry{
            Timestamp: e.Timestamp,
            Source:    "log",
            Agent:     e.Agent,
            Action:    e.Action,
            Task:      task,
            Detail:    e.Detail,
        }
        m.activities = appendActivity(m.activities, entry)
    }
    return m, nil
```

#### tea.WindowSizeMsg Handler

Updates layout dimensions and rebuilds styles:

```go
case tea.WindowSizeMsg:
    m.width = msg.Width
    m.height = msg.Height
    m.columnTier = ColumnTierForWidth(msg.Width)
    m.styles = NewStyles(msg.Width)
    return m, nil
```

#### errMsg Handler

Logs async errors (non-fatal):

```go
case errMsg:
    // No user-visible action for now; future: show in status bar
    return m, nil
```

#### watcherClosedMsg Handler

Falls back to tick-only refresh:

```go
case watcherClosedMsg:
    m.watcher = nil  // prevent re-subscribe attempts
    return m, nil
```

#### Default Handler

Unhandled messages pass through:

```go
default:
    return m, nil
```

#### Helper: appendActivity

Caps the activity buffer at 200 entries per spec:

```go
// appendActivity appends an entry to the activity slice, capping at 200 entries.
func appendActivity(activities []ActivityEntry, entry ActivityEntry) []ActivityEntry {
    activities = append(activities, entry)
    if len(activities) > 200 {
        activities = activities[len(activities)-200:]
    }
    return activities
}
```

#### Helper: formatAnomalyDetails

Converts `map[string]any` to a display string:

```go
// formatAnomalyDetails converts an anomaly's Details map to a compact display string.
func formatAnomalyDetails(details map[string]any) string
```

Sorts keys, formats as `key=value` pairs separated by spaces.

### Scope
- **In scope:** `internal/tui/update.go` (new file) — Update() with all data message handlers, appendActivity helper, formatAnomalyDetails helper
- **Out of scope:** Key dispatch (Phase 4), command execution (Phase 4), input mode handling (Phase 4), alert banner dismiss on keypress (Phase 4), cmdResult expiry clearing (Phase 4)
