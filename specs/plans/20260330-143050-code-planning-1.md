## Task 1: Define result type and sentinel error in `internal/ops/await_verdict.go`

**Intent:** Establish the `AwaitVerdictResult` struct and `ErrBudgetExhausted` sentinel so subsequent tasks can reference them.

**Design:**

```go
// Verdict constants
const (
    VerdictApproved   = "APPROVED"
    VerdictRejected   = "REJECTED"
    VerdictNewAttempt  = "NEW_ATTEMPT"
    VerdictTerminal    = "TERMINAL"
    VerdictTimeout     = "TIMEOUT"
    VerdictAborted     = "ABORTED"
)

// ErrBudgetExhausted is returned when the iteration/review-cycle budget
// would be exceeded on rejection — the agent should exit normally.
var ErrBudgetExhausted = stderrors.New("budget exhausted: iteration or review-cycle limit reached")

type AwaitVerdictResult struct {
    Verdict       string            // One of the Verdict* constants
    Reason        string            // Rejection reason or terminal explanation
    ReviewerAgent string            // Agent ID that issued the verdict (empty if timeout/abort)
    TaskStatus    models.TaskStatus // Final observed task status
    Iteration     int               // Current iteration number (post-reclaim if rejected)
    Guidance      string            // Inline guidance for the agent on rejection (equivalent to prior_rejection.tmpl)
}
```

**Rationale:**
- Verdict constants prevent string typos across handler (CP2) and tests (CP3).
- `Guidance` field carries the inline rejection guidance so the MCP handler can return it directly to the agent — avoiding a second state read.
- `ErrBudgetExhausted` is a sentinel (not `PreconditionError`) because it's not a caller mistake — it's a valid budget-gate outcome the handler translates to a clear message.

**Scope:** `internal/ops/await_verdict.go` (new file — types and error only, no function body).

**done_when:** `AwaitVerdictResult` struct and `ErrBudgetExhausted` sentinel are defined in `internal/ops/await_verdict.go`; the file compiles (`go build ./internal/ops/...` succeeds).

**spec_ref:** Spec step 1 "Return: AwaitVerdictResult{Verdict, Reason, ReviewerAgent, TaskStatus, Iteration}" and "On ErrBudgetExhausted: return clear error message"

## Task 2: Implement precondition validation and ownership acquisition in `AwaitVerdict`

**Intent:** Implement the entry path of `AwaitVerdict` — parameter validation, task status check, agent authorization, and atomic ownership acquisition.

**Design:**

Function signature (matches spec exactly):
```go
func AwaitVerdict(projectRoot, taskID, agentID string, timeout time.Duration) (*AwaitVerdictResult, error)
```

**Precondition checks** (return `PreconditionError` on failure):
1. `taskID` and `agentID` non-empty
2. Task exists (via `readTaskState`)
3. Task status is in the "awaitable set": `resolver.SubmittedStatus(rolePair)`, `resolver.ReviewingStatus(rolePair)`, or (if declared) `resolver.PartiallyApprovedStatus(rolePair)`. Use `loadResolver` to get the pipeline resolver, then check membership.
4. Agent was the last submitter: scan `task.History` in reverse for the most recent `TaskEventSubmittedForReview` entry; verify `entry.Agent == agentID`.

**Ownership acquisition** (atomic via `bb.Modify`):
```go
bb.Modify(func(s *models.State) error {
    agent, ok := s.Agents[agentID]
    if !ok {
        return &errors.NotFoundError{Entity: "agent", ID: agentID}
    }
    agent.Status = models.AgentStatusWaiting
    agent.CurrentTask = &taskID
    // Do NOT set LeaseExpires — heartbeat goroutine manages it.
    // The supervisor's heartbeat is already running and will renew
    // the task lease because agent.CurrentTask is now set.
    s.Agents[agentID] = agent
    return nil
})
```

**Key design note — why ownership re-acquisition is safe:**
`SubmitForReview` sets `agent.CurrentTask = nil` and `agent.LeaseExpires = nil`. Between submit and await, the supervisor is blocked on `executeAgent()`, so no other supervisor can observe the gap and steal the task. AwaitVerdict atomically re-sets `CurrentTask = &taskID`, restoring heartbeat-driven lease renewal. The spec's architectural assumption "No submit_for_review changes" confirms this window is safe.

**Error types:**
- `PreconditionError` for validation failures (wrong status, wrong agent, missing task)
- `OperationalError` for infrastructure failures (state read, resolver load)

**Scope:** `internal/ops/await_verdict.go` — add function skeleton with preconditions and ownership; the function returns a placeholder error after ownership (event loop added in Task 3).

**done_when:** `AwaitVerdict` validates preconditions (empty IDs, missing task, wrong status, wrong agent) and acquires ownership atomically; calling it with an invalid task status returns `PreconditionError`; `go build ./internal/ops/...` succeeds.

**spec_ref:** Spec step 1 "Validate preconditions" and "Acquire ownership" sections

**depends_on:** Task 1 (needs `AwaitVerdictResult` type and `ErrBudgetExhausted`)

## Task 3: Implement budget gate in `AwaitVerdict`

**Intent:** After ownership acquisition, check whether the iteration/review-cycle budget would be exceeded on a hypothetical rejection — if so, release ownership and return `ErrBudgetExhausted` immediately.

**Design:**

```go
// Load config and compute limits
config := state.Config
iterLimit := effectiveCoderIterationLimit(task, config)
reviewLimit := effectiveReviewCycleLimit(config)

// Simulate what happens on rejection
escalation, shouldEscalate := classifyLimitEscalation(
    task.ReviewCycles, reviewLimit,
    task.Iteration, iterLimit,
    task.AttemptNum,
)
if shouldEscalate {
    // Release ownership before returning
    releaseOwnership(bb, agentID) // helper: sets CurrentTask=nil, keeps WAITING
    return nil, ErrBudgetExhausted
}
```

**`releaseOwnership` helper** (reused by all exit paths):
```go
func releaseOwnership(bb *db.Blackboard, agentID string) error {
    return bb.Modify(func(s *models.State) error {
        if agent, ok := s.Agents[agentID]; ok {
            agent.CurrentTask = nil
            s.Agents[agentID] = agent
        }
        return nil
    })
}
```

This helper only clears `CurrentTask`. It does NOT change `Status` (remains WAITING) or `LeaseExpires` — the supervisor's `resetAgentAfterExit` handles those when the CLI session ends.

**Rationale for budget gate position:**
The budget gate runs AFTER ownership acquisition but BEFORE the watch loop. This prevents the agent from blocking for up to 25 minutes only to discover it can't iterate. The gate simulates rejection: if `classifyLimitEscalation` says `shouldEscalate=true`, the agent would be blocked or transitioned to a new attempt anyway — better to fail fast.

**Scope:** `internal/ops/await_verdict.go` — add budget gate logic between ownership acquisition and the event loop placeholder.

**done_when:** `AwaitVerdict` returns `ErrBudgetExhausted` when iteration or review-cycle limits are at capacity; ownership is released (agent.CurrentTask is nil) before returning the error; `go build ./internal/ops/...` succeeds.

**spec_ref:** Spec step 1 "Budget gate" section

**depends_on:** Task 2 (needs precondition + ownership code in place)

## Task 4: Implement event loop and result mapping in `AwaitVerdict`

**Intent:** Implement the watch loop that blocks until a verdict arrives, and map the final task status to the appropriate `AwaitVerdictResult`.

**Design — Event Loop:**

Follows the `waitForWorkEventDriven` pattern from `internal/agent/waitforwork.go`:

```go
watcher, err := bb.WatchForChanges()
if err != nil {
    // Fallback to polling (5s interval per spec)
    return awaitVerdictPolling(bb, taskID, agentID, timeout, resolver, rolePair, task)
}
defer watcher.Close()

deadline := time.Now().Add(timeout)
deadlineTimer := time.NewTimer(time.Until(deadline))
defer deadlineTimer.Stop()

abortTicker := time.NewTicker(1 * time.Second)
defer abortTicker.Stop()

for {
    select {
    case <-ctx.Done():
        releaseOwnership(bb, agentID)
        return nil, ctx.Err()

    case <-abortTicker.C:
        state, err := bb.ReadCached()
        if err != nil {
            continue
        }
        if state.Config.Mode == models.SystemModeStopped {
            releaseOwnership(bb, agentID)
            return &AwaitVerdictResult{Verdict: VerdictAborted, TaskStatus: task.Status}, nil
        }

    case <-watcher.Events():
        state, err := bb.ReadCached()
        if err != nil {
            continue
        }
        // Check abort first
        if state.Config.Mode == models.SystemModeStopped {
            releaseOwnership(bb, agentID)
            return &AwaitVerdictResult{Verdict: VerdictAborted, TaskStatus: task.Status}, nil
        }
        task := state.FindTask(taskID)
        if task == nil {
            releaseOwnership(bb, agentID)
            return &AwaitVerdictResult{Verdict: VerdictTerminal, Reason: "task disappeared"}, nil
        }
        result := checkVerdictStatus(task, resolver, rolePair)
        if result != nil {
            return handleVerdictResult(bb, result, task, agentID, projectRoot, resolver, rolePair)
        }
        // Still in submitted/reviewing/partially-approved — keep waiting

    case err := <-watcher.Errors():
        log.Printf("Watcher error, falling back to polling: %v", err)
        watcher.Close()
        return awaitVerdictPolling(bb, taskID, agentID, timeout, resolver, rolePair, task)

    case <-deadlineTimer.C:
        releaseOwnership(bb, agentID)
        return &AwaitVerdictResult{Verdict: VerdictTimeout, TaskStatus: task.Status}, nil
    }
}
```

**`checkVerdictStatus` helper** — determines if task has left the awaitable set:
```go
func checkVerdictStatus(task *models.Task, resolver *pipeline.Resolver, rolePair string) *verdictCheck {
    submitted := resolver.SubmittedStatus(rolePair)
    reviewing := resolver.ReviewingStatus(rolePair)
    partiallyApproved, _ := resolver.PartiallyApprovedStatus(rolePair) // may not exist

    switch task.Status {
    case submitted, reviewing:
        return nil // still waiting
    case partiallyApproved:
        if partiallyApproved != "" {
            return nil // quorum not met, keep waiting
        }
    }
    // Status has changed — verdict is in
    return &verdictCheck{status: task.Status}
}
```

**`handleVerdictResult`** — maps final status to result:
```go
func handleVerdictResult(...) (*AwaitVerdictResult, error) {
    approved := resolver.ApprovedStatus(rolePair)
    rejected := resolver.RejectedStatus(rolePair)

    switch {
    case task.Status == approved:
        releaseOwnership(bb, agentID)
        reviewer := extractReviewerFromHistory(task)
        return &AwaitVerdictResult{
            Verdict: VerdictApproved, TaskStatus: task.Status,
            ReviewerAgent: reviewer,
        }, nil

    case task.Status == rejected:
        reviewer := extractReviewerFromHistory(task)
        reason := ""
        if task.RejectionReason != nil {
            reason = *task.RejectionReason
        }

        // Attempt auto-reclaim via ops.ClaimTask
        claimResult, err := ClaimTask(projectRoot, taskID, agentID)
        if err != nil {
            var pe *PreconditionError
            if stderrors.As(err, &pe) {
                // ClaimTask refused — check if it's a limit escalation
                if strings.Contains(pe.Reason, "new attempt") {
                    releaseOwnership(bb, agentID)
                    return &AwaitVerdictResult{
                        Verdict: VerdictNewAttempt, Reason: reason,
                        ReviewerAgent: reviewer, TaskStatus: task.Status,
                    }, nil
                }
                // Blocked or other limit
                releaseOwnership(bb, agentID)
                return &AwaitVerdictResult{
                    Verdict: VerdictTerminal, Reason: reason,
                    ReviewerAgent: reviewer, TaskStatus: task.Status,
                }, nil
            }
            // Infrastructure error
            releaseOwnership(bb, agentID)
            return nil, fmt.Errorf("auto-reclaim failed: %w", err)
        }

        // Reclaim succeeded — agent continues in same session
        guidance := buildRejectionGuidance(reason, task)
        return &AwaitVerdictResult{
            Verdict: VerdictRejected, Reason: reason,
            ReviewerAgent: reviewer, TaskStatus: task.Status,
            Iteration: task.Iteration,
            Guidance: guidance,
        }, nil

    default:
        // BLOCKED, SUPERSEDED, INTEGRATION_FAILED, etc.
        releaseOwnership(bb, agentID)
        return &AwaitVerdictResult{
            Verdict: VerdictTerminal, TaskStatus: task.Status,
            Reason: fmt.Sprintf("task entered terminal status: %s", task.Status),
        }, nil
    }
}
```

**`awaitVerdictPolling` fallback** — same logic but with `time.Ticker` (5s interval per spec) instead of fsnotify:
```go
func awaitVerdictPolling(bb, taskID, agentID string, timeout time.Duration, ...) (*AwaitVerdictResult, error) {
    deadline := time.Now().Add(timeout)
    ticker := time.NewTicker(5 * time.Second)
    defer ticker.Stop()
    for {
        select {
        case <-ticker.C:
            // same check logic as event-driven path
        case <-deadlineTimer.C:
            releaseOwnership(...)
            return timeout result
        }
    }
}
```

**`buildRejectionGuidance` helper:**
Constructs inline text equivalent to `prior_rejection.tmpl` content:
- "MUST ADDRESS the following rejection feedback: ..."
- Includes scope_extensions hint for coders
- Includes updated iteration/remaining budget info

**`extractReviewerFromHistory` helper:**
Scans `task.History` in reverse for the most recent `TaskEventReviewVerdictApproved` or `TaskEventReviewVerdictRejected` entry; returns `*entry.Agent`.

**Key design notes:**
1. **ABORT detection uses `state.Config.Mode == SystemModeStopped`** — this matches the existing pattern in `waitForWorkEventDriven` (`isSystemStopped`). The spec mentions `paths.AbortPath` but no such path exists in the codebase; the state-based check is the canonical mechanism.
2. **PartiallyApproved keeps waiting** — per spec, quorum not yet met means review is still in progress.
3. **Auto-reclaim delegates to `ClaimTask`** — per spec, "ClaimTask internally checks limits via `classifyLimitEscalation`" so AwaitVerdict doesn't need its own same-vs-new-attempt detection. The budget gate (Task 3) is a pre-check; ClaimTask is the authoritative decision.
4. **Context parameter**: The function needs a `context.Context` for cancellation. Add it as the first parameter: `func AwaitVerdict(ctx context.Context, projectRoot, taskID, agentID string, timeout time.Duration)`. The MCP handler (CP2 scope) will pass `context.Background()` or a derived context.

**Scope:** `internal/ops/await_verdict.go` — event loop, polling fallback, `checkVerdictStatus`, `handleVerdictResult`, `releaseOwnership`, `buildRejectionGuidance`, `extractReviewerFromHistory`.

**done_when:** `AwaitVerdict` compiles with full event loop, polling fallback, and all verdict mappings (APPROVED, REJECTED with auto-reclaim, NEW_ATTEMPT, TERMINAL, TIMEOUT, ABORTED); `go build ./internal/ops/...` succeeds.

## Task 6: Heartbeat continuity verification

**Intent:** Verify that the supervisor heartbeat keeps running while the agent blocks on the MCP tool call, ensuring task and agent leases don't expire during the await.

**Design — Verification approach:**

This is a verification task, not a code change. The verification confirms the spec's architectural assumption: "The supervisor's heartbeat goroutine runs for the supervisor process lifetime, not per-agent-session."

**Evidence chain:**

1. **Heartbeat start**: In `supervisor.go`, `RunSupervisor()` starts the heartbeat goroutine BEFORE the main work loop:
   ```go
   go func() { hb.Start(heartbeatCtx) }()
   ```
   The `heartbeatCtx` is derived from the supervisor's top-level context, cancelled only when `RunSupervisor()` exits.

2. **Blocking during MCP call**: When the agent calls `liza_await_verdict`, the CLI process is running (the agent session is active). The supervisor is blocked on `executeAgent()` (in `systemctl.go`), which blocks on `config.Executor.Execute()`. The heartbeat goroutine is NOT blocked — it runs independently.

3. **Task lease renewal**: In `heartbeat.go`, `beat()` checks `agent.CurrentTask != nil`. When AwaitVerdict sets `CurrentTask = &taskID`, the heartbeat will renew the task's `LeaseExpires` on every beat (default 60s interval, 30min lease). This prevents lease expiration during the up-to-25-minute wait.

4. **Agent lease renewal**: The heartbeat also renews `agent.LeaseExpires`, keeping the agent registered.

5. **Failure mode**: If heartbeat were changed to require agent-side signaling (e.g., checking a channel), the supervisor process would still be alive but the lease would expire. The AwaitVerdict function runs in the MCP handler (agent-side), so it cannot participate in supervisor-side heartbeat. This is the spec's documented risk: "If heartbeat were changed to require agent-side signaling, this would break."

**Verification artifact:** The plan file itself (this document) serves as the verification artifact. No code changes are needed.

**Scope:** Verification only — `internal/agent/heartbeat.go`, `internal/agent/supervisor.go`, `internal/agent/systemctl.go` (read-only).

**done_when:** Plan documents the evidence chain confirming heartbeat continuity: (1) heartbeat goroutine lifecycle is independent of agent session, (2) `CurrentTask` set by AwaitVerdict triggers task lease renewal in heartbeat, (3) no agent-side signaling is required.

**spec_ref:** Spec "Architectural Assumptions — Heartbeat continuity" and "`internal/agent/heartbeat.go` — Verify — supervisor heartbeat keeps lease alive during block"
