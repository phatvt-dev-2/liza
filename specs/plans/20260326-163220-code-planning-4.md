**Task:** code-planning-4

## Overview

Implement the interactivity layer for the TUI dashboard. This phase adds:

1. **Key dispatch** ŌĆö `tea.KeyMsg` handling in `Update()` with mode-sensitive routing (normal, inline, form)
2. **Command execution** ŌĆö Bubbletea `Cmd` functions that call `commands.*Command()` directly or spawn processes via `exec.Command`
3. **Inline input (Tier 1)** ŌĆö Bubbles `textinput` for spawn role (with tab-completion) and pause reason
4. **Huh form overlay (Tier 2)** ŌĆö 6-field form for add-task, composed into the Bubbletea model

All actions flow through Bubbletea's message-passing architecture: key events dispatch to mode-specific handlers, action Cmds run in goroutines, and result messages update the model.

## Task 2: Action Cmd functions (commands.go)

### Intent
Implement Bubbletea `Cmd` functions for all 7 user-triggered actions (spawn, pause, resume, checkpoint, stop, add-task, load-roles). Each Cmd runs in a goroutine and returns a `CmdResultMsg` or a specialized message.

### Design

#### spawnAgentCmd ŌĆö exec.Command (background process)

```go
// spawnAgentCmd spawns a new agent process for the given role.
// Uses exec.Command("liza", "agent", role).Start() per spec ┬¦Keybindings.
// The child process is detached (background, own PTY).
// Returns CmdResultMsg with success/error status.
func spawnAgentCmd(projectRoot, role string) tea.Cmd
```

Implementation:
1. Build `exec.Command("liza", "agent", role)` with `Dir: projectRoot`
2. Set `SysProcAttr` for process group detachment (`Setpgid: true`)
3. Call `cmd.Start()` (non-blocking ŌĆö child runs in background)
4. Return `CmdResultMsg{Success: true, Message: "Spawned " + role}` or error

**Note:** The child process inherits the TUI's stdout/stderr, which would corrupt the display. Set `cmd.Stdout` and `cmd.Stderr` to `os.DevNull` (or to agent log files) to prevent this. The agent supervisor manages its own logging.

#### pauseSystemCmd ŌĆö commands.PauseCommand direct call

```go
// pauseSystemCmd pauses the system with an optional reason.
// Calls commands.PauseCommand() directly (same process, no subprocess).
// Returns CmdResultMsg with result.
func pauseSystemCmd(projectRoot, reason string) tea.Cmd
```

Implementation: calls `commands.PauseCommand(projectRoot, reason, "operator")`. Captures stdout output from PauseCommand is irrelevant (it prints to stdout, which would corrupt TUI); instead, the Cmd returns a `CmdResultMsg` based on the error return. The `commands.*Command` functions print to stdout ŌĆö since the TUI owns stdout, these prints would corrupt the display. The Cmd wraps the call and only uses the error return to build `CmdResultMsg`. The printed output is harmless during the Cmd goroutine since Bubbletea suspends rendering during Cmd execution ŌĆö but for correctness, a future refinement could redirect stdout. For Phase 4, relying on error status is sufficient.

**Alternative approach (cleaner):** Call `ops.Pause()` directly instead of `commands.PauseCommand()`, since the `commands.*` layer adds stdout printing which the TUI doesn't need. The TUI gets feedback via `CmdResultMsg` instead.

Decision: Call `ops.*` functions directly (Pause, Resume, Stop, SprintCheckpoint, AddTask) to avoid stdout corruption. The `commands.*` layer is the CLI-facing wrapper; the TUI bypasses it.

#### resumeSystemCmd ŌĆö ops.Resume direct call

```go
// resumeSystemCmd resumes the system.
// Calls ops.Resume() directly.
// Returns CmdResultMsg with result.
func resumeSystemCmd(projectRoot string) tea.Cmd
```

#### checkpointCmd ŌĆö ops.SprintCheckpoint direct call

```go
// checkpointCmd creates a sprint checkpoint.
// Calls ops.SprintCheckpoint() directly.
// Returns CmdResultMsg with result.
func checkpointCmd(projectRoot string) tea.Cmd
```

#### stopSystemCmd ŌĆö ops.Stop direct call

```go
// stopSystemCmd stops the system, then signals the TUI to quit.
// Calls ops.Stop() directly.
// Returns stopDoneMsg on success (triggers tea.Quit in Update).
// Returns CmdResultMsg with error on failure.
func stopSystemCmd(projectRoot string) tea.Cmd
```

On success: returns `stopDoneMsg{}` (Update handler triggers `tea.Quit`).
On failure: returns `CmdResultMsg{Success: false, Message: err.Error()}`.

#### addTaskCmd ŌĆö ops.AddTask direct call

```go
// addTaskCmd adds a new task from Huh form input.
// Calls ops.AddTask() directly.
// Returns CmdResultMsg with result.
func addTaskCmd(projectRoot string, input *commands.TaskInput) tea.Cmd
```

Implementation: resolves statePath and logPath from `paths.New(projectRoot)`, converts `commands.TaskInput` to `ops.AddTaskInput`, calls `ops.AddTask()`.

#### loadRolesCmd ŌĆö pipeline.Resolver discovery

```go
// loadRolesCmd loads role names from pipeline config for tab-completion.
// Returns rolesMsg with sorted role names.
// Returns empty rolesMsg if config not found (non-fatal).
func loadRolesCmd(projectRoot string) tea.Cmd
```

Implementation: calls `ops.LoadResolverForModels(projectRoot)`, then `resolver.AllRoleNames()`. Returns `rolesMsg{Roles: names}`. On error (no pipeline config), returns `rolesMsg{Roles: nil}` ŌĆö tab-completion simply won't work.

### Scope
- **In scope:** `internal/tui/commands.go` ŌĆö add 7 new Cmd functions: `spawnAgentCmd`, `pauseSystemCmd`, `resumeSystemCmd`, `checkpointCmd`, `stopSystemCmd`, `addTaskCmd`, `loadRolesCmd`
- **Out of scope:** Update handlers for returned messages (Tasks 3-5), model field additions (Task 1), view changes

### Done When
- `internal/tui/commands.go` compiles with all 7 new Cmd functions
- `spawnAgentCmd(projectRoot, role)` returns a `tea.Cmd` that calls `exec.Command("liza", "agent", role).Start()` with stdout/stderr redirected to `os.DevNull`, `SysProcAttr.Setpgid: true`, and returns `CmdResultMsg`
- `pauseSystemCmd(projectRoot, reason)` returns a `tea.Cmd` that calls `ops.Pause(projectRoot, reason, "operator")` and returns `CmdResultMsg`
- `resumeSystemCmd(projectRoot)` returns a `tea.Cmd` that calls `ops.Resume(projectRoot, "operator")` and returns `CmdResultMsg`
- `checkpointCmd(projectRoot)` returns a `tea.Cmd` that calls `ops.SprintCheckpoint(projectRoot, "")` and returns `CmdResultMsg`
- `stopSystemCmd(projectRoot)` returns a `tea.Cmd` that calls `ops.Stop(projectRoot, "TUI stop", "operator")` and returns `stopDoneMsg` on success or `CmdResultMsg` on error
- `addTaskCmd(projectRoot, input)` returns a `tea.Cmd` that calls `ops.AddTask()` and returns `CmdResultMsg`

## Task 3: Key dispatch ŌĆö normal mode, simple commands, and quit (update.go)

**Depends on:** Task 1 (model fields: `InlineAction`, `textInput`, `inlineLabel`, `inlineAction`), Task 2 (Cmd functions: `resumeSystemCmd`, `checkpointCmd`, `loadRolesCmd`)

### Design

#### tea.KeyMsg Handler (added to Update type switch)

```go
case tea.KeyMsg:
    // Dismiss alert banner on any keypress (spec ┬¦Alert Banner)
    if m.alertBanner != nil {
        m.alertBanner = nil

    // Route to mode-specific handler
```

```go
// handleNormalKey dispatches key events in normal mode.
```

```go
    }
    return m, nil
}
```

### Intent

### Design

#### handleInlineKey ŌĆö full implementation

```go
// handleInlineKey handles key events in inline input mode.
// Delegates to textinput for character input. Handles Tab (completion),
// Enter (confirm action), and Esc (cancel) specially.
func (m Model) handleInlineKey(msg tea.KeyMsg) (tea.Model, tea.Cmd)
```

```go

case key.Matches(msg, key.NewBinding(key.WithKeys("esc"))):
    // Cancel: return to normal mode

    m.inlineAction = InlineActionNone
    m.textInput.Blur()

    value := m.textInput.Value()

    m.inlineAction = InlineActionNone

    return m, nil

    var cmd tea.Cmd
    m.textInput, cmd = m.textInput.Update(msg)
    // Reset completion state when text changes (user typed something)
    m.completionIdx = 0
    m.completionPrefix = ""
    return m, cmd
```

#### cycleCompletion ŌĆö Tab-completion for roles

```go
// cycleCompletion cycles through role names matching the current input prefix.
func (m Model) cycleCompletion() Model
```

Implementation:
1. If `m.roleCompletions` is empty, return unchanged (no completions available)
2. If `m.completionPrefix` is empty (first Tab press), capture current `textInput.Value()` as `m.completionPrefix`
3. Filter `m.roleCompletions` to those with prefix matching `m.completionPrefix` (case-insensitive). If prefix is empty, use all roles.
4. If no matches, return unchanged
5. Select `matches[m.completionIdx % len(matches)]`
6. Set `m.textInput.SetValue(selected)`
7. Increment `m.completionIdx`

When the user types (not Tab), the `default` case resets `completionIdx` and `completionPrefix` so the next Tab starts fresh.

```go
func (m Model) handleFormKey(msg tea.KeyMsg) (tea.Model, tea.Cmd) {
    // Esc cancels the form
    if msg.String() == "esc" {
        m.inputMode = InputModeNormal
        m.huhForm = nil
        return m, nil
    }

    // Delegate to Huh form
    form, cmd := m.huhForm.Update(msg)
    m.huhForm = form.(*huh.Form)

    // Check if form completed (submitted)
    if m.huhForm.State == huh.StateCompleted {
        m.inputMode = InputModeNormal
        input := m.extractFormData()
        m.huhForm = nil
        return m, addTaskCmd(m.projectRoot, input)
    }

    // Check if form was aborted
    if m.huhForm.State == huh.StateAborted {
        m.inputMode = InputModeNormal
        m.huhForm = nil
        return m, nil
    }

    return m, cmd
}
```
