## Scope Extension

**File:** `internal/commands/init_test.go` (lines 900-911)
**Justification:** The test at lines 900-911 asserts that `mcp__liza__*` permission entries are present in the embedded settings generated by `liza init`. Removing these entries from `internal/embedded/claude-settings.json` (in-scope) causes this assertion to fail, making `go test ./internal/commands/...` fail. The assertion removal is a mechanical consequence of the template content change and must be atomic with it to satisfy the parent task's done_when (`go test ./internal/commands/... passes`).

**Overlap with sibling Scope 1:** Scope 1 (architecture-3-architecture-to-code-plan-1) lists `init_test.go` in its scope for "remove mcp__liza__* permission assertions." When both tasks merge, git will auto-resolve the identical deletion. Scope 1 also plans to remove `.mcp.json` fixtures from init_test.go, but grep confirms no `.mcp.json` references exist in init_test.go — that planned change is a no-op.

### init_test.go (scope extension)

3. **Remove lines 900-911:** The assertion block that checks for `mcp__liza__*` entries in the allow array:

**Scope:** Modify `internal/embedded/claude-settings.json` (remove line 36, remove lines 44-69, remove blank line 70). Modify `internal/commands/init_test.go` (remove lines 900-911, scope extension — see justification above).
