# Code Plan: Many-to-One Transition Engine

**Task:** code-planning-2
**Spec:** specs/goals/20260405-architecture-step.md
**Scope:** Many-to-one transition cardinality engine: cohort detection in proceed.go (sibling parent_task grouping, status validation), child creation from N parents, deterministic child-ID scheme, spec_ref inheritance, transitions_executed bookkeeping across cohort, crash recovery, pipeline/resolver.go introspection methods, and wake detection in advance_sprint.go/workdetection.go. Covers the engine portion of spec item 2. Excludes task model changes (CP1), arch_ref (CP3), context sections and prompts (CP4).

## Context

The architecture step introduces a `many-to-one` transition cardinality: N approved tasks fan in to create 1 child task. This is the engine implementation that makes it work at runtime. CP1 handles the foundational pieces (roles, pipeline YAML, `ParentTasks` field, validation-only cardinality acceptance). This plan builds the behavioral interpretation of `many-to-one` in the transition engine.

### Key Codebase Observations

1. **`proceedInner` is the core transition dispatcher**: Lines 129-198 of `proceed.go` handle both manual (`Proceed`) and auto (`ExecuteAvailableTransitions`) paths. A switch on `tDef.cardinality` routes to per-subtask or one-to-one logic. The default case returns an error for unsupported cardinalities — `many-to-one` currently falls through here (CP1 adds the YAML validation but no engine behavior).

2. **Two execution paths converge on `proceedInner`**: Manual `Proceed` (CLI, sets `requiredStatus` from the transition `from` ref) and `ExecuteAvailableTransitions` (supervisor, overrides `requiredStatus` to MERGED at `proceed.go:578`). For many-to-one, cohort members may be at the approved status OR at MERGED, depending on path. Both must be accepted.

3. **`transitions_executed` is per-source-task**: Currently `task.TransitionsExecuted[transitionName] = true` is set on the single source task (proceed.go:166). For many-to-one, this must be set on ALL N cohort members for idempotency. This also makes `ExecuteAvailableTransitions` naturally dedup: after the first cohort member fires the transition and marks all members, subsequent members' `AvailableManualTransitions` won't return the transition (resolver.go:488-489 checks `transitionsExecuted`).

4. **`recoverCrashedTransition` checks child existence per source task**: Lines 200-261 switch on cardinality. Many-to-one crash recovery must check whether the single child exists, and whether all cohort members have `transitions_executed` set.

5. **Deterministic child IDs follow `<parent>-<transition>[-<index>]` pattern**: `perSubtaskChildID` (line 26) and `oneToOneChildID` (line 30). Many-to-one uses a similar pattern: `<cohort-shared-parent>-<transition-name>`, since all cohort members share a parent task (created via per-subtask from the same upstream task).

6. **`IsUnconsumedPlanningOutput` drives PLANNING_COMPLETE wake detection**: `advance_sprint.go:212-219` checks for MERGED status + non-empty output + no transitions executed. US tasks approved for many-to-one don't match this pattern (no output[]). A new detection mechanism is needed.

7. **`DetectOrchestratorWakeTriggers` checks for terminal tasks first**: `workdetection.go:94` — the many-to-one-ready check belongs inside the `AllPlannedTasksTerminal` block, alongside PLANNING_COMPLETE, since the cohort becomes complete only when all members are merged (terminal).

8. **Resolver lacks transition iteration and source role-pair extraction**: `resolver.go` has `Transition(name)` but no way to iterate all transitions. `TransitionTargetRolePair` exists but no `TransitionSourceRolePair`. Both are needed for wake detection (find many-to-one transitions → extract source role-pair → detect cohort readiness).

9. **`computeInheritedDeps` needs many-to-one case**: Lines 865-895 iterate upstream deps and collect child task IDs by cardinality. Many-to-one creates one child per cohort, so multiple upstream deps in the same cohort produce the same child ID — dedup is required.

10. **Sprint carry-forward for many-to-one**: `collectMergedPlanningWithUnconsumedOutput` (advance_sprint.go:300-309) carries merged planning tasks with unfired transitions. MERGED many-to-one cohort members with unfired transitions need the same carry-forward treatment to survive sprint advance.

## Design Decisions

### D1: Delegate many-to-one to dedicated function in proceedInner

Rather than adding cohort logic to the existing switch cases, `proceedInner` delegates to `proceedManyToOneInner` when `tDef.cardinality == "many-to-one"`. This keeps the existing per-subtask and one-to-one paths unchanged and isolates the cohort detection, multi-parent child creation, and multi-task transitions_executed bookkeeping.

### D2: Cohort = siblings sharing a parent_task with the same role_pair

The spec defines cohort as "sibling tasks sharing a parent_task." Siblings are created via per-subtask from the same upstream task, so they share a parent. The algorithm: given a trigger task, find all tasks in state that share at least one parent (via `EffectiveParentTasks()`) AND have the same `RolePair`. This is sufficient because per-subtask children always share the same single parent.

### D3: Accept both approved status and MERGED for cohort members

The manual `Proceed` path sets `requiredStatus` from the transition's `from` ref (e.g., `US_APPROVED`). The `ExecuteAvailableTransitions` path overrides to `MERGED`. Cohort members may be at either state. `proceedManyToOneInner` accepts both: `member.Status == tDef.requiredStatus || member.Status == models.TaskStatusMerged`.

### D4: Child ID from cohort's shared parent + transition name

`manyToOneChildID(cohortParentID, transitionName) = "<cohortParentID>-<transitionName>"`. The cohort's shared parent (e.g., the epic plan task) is deterministic and unique per cohort. Multiple many-to-one transitions from the same cohort (if any) are distinguished by transition name.

### D5: Many-to-one wake detection alongside PLANNING_COMPLETE

The new `WakeTriggerManyToOneReady` trigger is checked inside the `AllPlannedTasksTerminal` block in `DetectOrchestratorWakeTriggers`, immediately after the PLANNING_COMPLETE check. This is correct because the cohort becomes complete when all members reach terminal state (MERGED). A new ops helper `CountReadyManyToOneCohorts` encapsulates the detection logic.

### D6: Carry-forward for unfired many-to-one cohorts

Merged cohort members with unfired many-to-one transitions must survive sprint advance. A new `collectMergedManyToOneWithUnfiredTransition` function in `advance_sprint.go` mirrors the existing `collectMergedPlanningWithUnconsumedOutput` pattern: iterate sprint scope, find MERGED tasks that are part of a complete cohort with no `transitions_executed` for the relevant transition.

func (r *Resolver) TransitionSourceRolePair(name string) (string, error) {
    t, err := r.Transition(name)
    if err != nil {
        return "", err

    // Try 3-part ref first (pipeline-transitions).
    _, fromPair, _, err3 := parse3PartRef(t.From)

    return fromPair2, nil

```go
// findManyToOneCohort finds all sibling tasks that form a many-to-one cohort
// with the trigger task. Siblings share a parent_task and the same role_pair.
// Returns the cohort members (as pointers into state.Tasks) and the shared parent ID.
func findManyToOneCohort(s *models.State, triggerTask *models.Task) ([]*models.Task, string, error) {
    parents := triggerTask.EffectiveParentTasks()
    if len(parents) == 0 {
        return nil, "", fmt.Errorf("trigger task %q has no parent_task — cannot determine many-to-one cohort", triggerTask.ID)
    }
    sharedParentID := parents[0]

    var cohort []*models.Task
    for i := range s.Tasks {
        task := &s.Tasks[i]
        if task.RolePair != triggerTask.RolePair {
            continue
        }
        taskParents := task.EffectiveParentTasks()
        if slices.Contains(taskParents, sharedParentID) {
            cohort = append(cohort, task)
        }
    }

    if len(cohort) == 0 {
        return nil, "", fmt.Errorf("no cohort members found for trigger task %q", triggerTask.ID)
    }

    // Sort for deterministic ordering
    slices.SortFunc(cohort, func(a, b *models.Task) int {
        return strings.Compare(a.ID, b.ID)
    })

    return cohort, sharedParentID, nil
}
```

Add `proceedManyToOneInner`:
```go
// proceedManyToOneInner handles many-to-one transition logic. The trigger task
// identifies the cohort (all siblings sharing a parent_task with the same role_pair).
// The transition fires only when ALL cohort members are at the required status
// (or MERGED, for the ExecuteAvailableTransitions path). Creates one child task
// linked to all N cohort members and sets transitions_executed on all of them.
func proceedManyToOneInner(s *models.State, taskID, transitionName string, tDef transitionDef, inheritedDeps []string, now time.Time, result *ProceedResult) error {
    task := s.FindTask(taskID)
    if task == nil {
        return fmt.Errorf("task %q not found", taskID)
    }

    // Accept both the required status and MERGED (see Design Decision D3)
    if task.Status != tDef.requiredStatus && task.Status != models.TaskStatusMerged {
        return fmt.Errorf("task %q must be at %s (or MERGED) for many-to-one transition %q (current: %s)",
            taskID, tDef.requiredStatus, transitionName, task.Status)
    }

    cohort, sharedParentID, err := findManyToOneCohort(s, task)
    if err != nil {
        return fmt.Errorf("many-to-one cohort detection failed: %w", err)
    }

    // Validate all cohort members are ready
    for _, member := range cohort {
        if member.Status != tDef.requiredStatus && member.Status != models.TaskStatusMerged {
            return fmt.Errorf("many-to-one cohort incomplete: member %q is at %s (need %s or MERGED)",
                member.ID, member.Status, tDef.requiredStatus)
        }
    }

    childID := manyToOneChildID(sharedParentID, transitionName)

    // Idempotency: check if any cohort member already has transition executed
    anyExecuted := false
    for _, member := range cohort {
        if member.TransitionsExecuted[transitionName] {
            anyExecuted = true
            break
        }
    }

    if anyExecuted {
        // Crash recovery: check if child exists
        if existing := s.FindTask(childID); existing != nil {
            // Ensure all cohort members have transitions_executed set (repair partial)
            for _, member := range cohort {
                if member.TransitionsExecuted == nil {
                    member.TransitionsExecuted = make(map[string]bool)
                }
                member.TransitionsExecuted[transitionName] = true
            }
            patchInheritedDeps(existing, inheritedDeps)
            return fmt.Errorf("%w: %q on cohort (parent %s)", errTransitionAlreadyExecuted, transitionName, sharedParentID)
        }
        // Child missing — fall through to create it (crash recovery)
    }

    // Set transitions_executed on ALL cohort members BEFORE child creation (crash recovery)
    for _, member := range cohort {
        if member.TransitionsExecuted == nil {
            member.TransitionsExecuted = make(map[string]bool)
        }
        member.TransitionsExecuted[transitionName] = true
    }

    child := buildManyToOneChild(childID, cohort, sharedParentID, tDef, inheritedDeps, now)
    s.Tasks = append(s.Tasks, child)
    result.ChildTaskIDs = append(result.ChildTaskIDs, childID)

    // Populate CohortTaskIDs
    cohortIDs := make([]string, len(cohort))
    for i, member := range cohort {
        cohortIDs[i] = member.ID
    }
    result.CohortTaskIDs = cohortIDs

    // Record history on the trigger task
    task.History = append(task.History, models.TaskHistoryEntry{
        Time:  now,
        Event: models.TaskEventTransitionExecuted,
        Extra: map[string]any{
            "transition":   transitionName,
            "cardinality":  "many-to-one",
            "cohort_size":  len(cohort),
            "cohort_parent": sharedParentID,
            "children":     1,
        },
    })

    return nil
}
```

Add `IsManyToOneReady`:
```go
// IsManyToOneReady checks if a MERGED task is part of a complete many-to-one
// cohort whose transition has not yet been executed. Used by both carry-forward
// and wake detection. A cohort is complete when all siblings (same parent, same
// role_pair) are MERGED with no transitions_executed for the transition name.
func IsManyToOneReady(task *models.Task, state *models.State, m2oTransitions []ManyToOneTransitionInfo) bool {
    if task == nil || task.Status != models.TaskStatusMerged {
        return false
    }
    parents := task.EffectiveParentTasks()
    if len(parents) == 0 {
        return false
    }

    for _, m2o := range m2oTransitions {
        if task.RolePair == "" {
            continue
        }
        // Check if this task's role-pair matches the transition's source
        // The task's role-pair must be looked up against the resolver data.
        // Since ManyToOneTransitionInfo has the source role-pair, compare directly.
        // But tasks store the role-pair (e.g., "us-writing-pair"), not the role name.
        // The m2o.SourceRolePair is exactly the role-pair name. Compare directly.
        if task.RolePair != m2o.SourceRolePair {
            continue
        }
        if task.TransitionsExecuted[m2o.Name] {
            continue
        }

        // Check if ALL siblings in the cohort are MERGED
        sharedParentID := parents[0]
        allReady := true
        for i := range state.Tasks {
            sibling := &state.Tasks[i]
            if sibling.RolePair != task.RolePair {
                continue
            }
            siblingParents := sibling.EffectiveParentTasks()
            if !slices.Contains(siblingParents, sharedParentID) {
                continue
            }
            if sibling.Status != models.TaskStatusMerged {
                allReady = false
                break
            }
            if sibling.TransitionsExecuted[m2o.Name] {
                allReady = false  // already executed — not "unfired"
                break
            }
        }
        if allReady {
            return true
        }
    }
    return false
}
```

// countReadyManyToOneCohorts counts distinct many-to-one cohorts that are
// ready for transition (all members MERGED, transition not yet executed).
// Uses ops.IsManyToOneReady and deduplicates by cohort (shared parent + transition).
func countReadyManyToOneCohorts(state *models.State, m2oTransitions []ops.ManyToOneTransitionInfo) int {
    // Track seen cohorts by (sharedParentID, transitionName) to avoid double-counting
    type cohortKey struct {
        parentID       string
        transitionName string
    }
    seen := make(map[cohortKey]bool)

    for _, taskID := range state.Sprint.Scope.Planned {
        task := state.FindTask(taskID)
        if !ops.IsManyToOneReady(task, state, m2oTransitions) {
            continue
        }
        parents := task.EffectiveParentTasks()
        if len(parents) == 0 {
            continue
        }
        for _, m2o := range m2oTransitions {
            if task.RolePair == m2o.SourceRolePair {
                key := cohortKey{parents[0], m2o.Name}
                seen[key] = true
            }
        }
    }
    return len(seen)
}
